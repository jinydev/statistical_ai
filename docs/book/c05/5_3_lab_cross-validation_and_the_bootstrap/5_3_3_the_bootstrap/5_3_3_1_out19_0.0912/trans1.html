The bootstrap approach can be used to assess the variability of the coefficient estimates and predictions from a statistical learning method. Here we use the bootstrap approach in order to assess the variability of the estimates for $\beta_0$ and $\beta_1$, the intercept and slope terms for the linear regression model that uses horsepower to predict mpg in the Auto data set. We will compare the estimates obtained using the bootstrap to those obtained using the formulas for $\text{SE}(\hat{\beta}_0)$ and $\text{SE}(\hat{\beta}_1)$ described in Section 3.1.2. 부트스트랩 접근 타진 방식 기조는 임의의 기계 통계 학습 메커니즘 구동 모델상에서 돌출돼 떨어져 나오는 각종 계수 추정 지표들(coefficient estimates) 및 예측 파생값 결괏치 본연이 지닌 가변성(variability) 널뛰기 이격을 진단 평가(assess)하려는 용도 기반으로 폭넓게 착취 운용(used to)될 수 있습니다. 작금의 대목(Here)에서 우리들은 바로 이 본연의 부트스트랩 접근 시도를 채택 취하여 활용할 셈인데, 이의 목적은 기 출시된 Auto 차량 제원 데이터 더미 묶음 정보 위에서 기어코 horsepower 인자를 축으로 mpg 결괏수치를 측량 견인 예측(predict)코자 도출 산출해 올렸던 일직선 1차 회귀 모듈 함수가 뿜어내고 산출한 구성요소 성분들 즉 $Y$ 절편 위치 축항 및 곡선 편향 기울기 단위체로 지목 추종되는 쌍방의 모수 표적 치수 $\beta_0$ 및 $\beta_1$ 두 녀석이 태생적으로 발산 웅거하고 지닌 편차적 변동성 이탈 변위를 직접 검정 평가(assess) 수리하려는 소임 때문입니다. 향후 우리는 방금 부트스트랩을 매개로 삼아 도출 획득 구득 달성된 개별 모수 쌍의 편차 추산치 수위 결과물을 예전 3.1.2장 도입부에 언급 기입 서술(described) 남겼었던 $\text{SE}(\hat{\beta}_0)$ 와 $\text{SE}(\hat{\beta}_1)$ 측량 공식 체계를 활용해 도출해 건졌던 당대 기록 산출물들(those)과 나란히 상호 대조 왈가왈부 비교(compare) 교차 점검할 것입니다.

To use our boot_SE() function, we must write a function (its first argument) that takes a data frame D and indices idx as its only arguments. But here we want to bootstrap a specific regression model, specified by a model formula and data. We start by writing a generic function boot_OLS() for bootstrapping a regression model that takes a formula to define the corresponding regression. We use the clone() function to make a copy of the formula that can be refit to the new dataframe. This means that any derived features such as those defined by poly() (which we will see shortly), will be re-fit on the resampled data frame. 그 시발점(start by)으로 우리는 해당 대응 회기 체계를 뚜렷이 설계 입증 확정(define)코자 그 뼈대 공식 자체를 입력 흡인(takes) 접수하는, 회귀 모형용 뺑뺑이 재추출 기믹 가동 공구로서의 일반 범용 형태성 성향(generic)의 다용도 함수인 boot_OLS() 를 백지부터 하나 기입 마련 축조하는 것에서부터 신호탄을 쏩니다. 우리들은 그 안에 내장된 파이썬 기능 공구 clone() 명령어 체제 기능을 활용 착취하여 본 공식 규격 본체와 구별되는 거푸집 복사본 사본 껍데기(copy)를 1차 마련해 복제시켜 냄으로써, 그것이 향후 계속 재등장할 낯설고 신규한 새 데이터 판 위에 족족 자유자재 재 장착 접합 피팅(refit) 구사되도록 활로를 열어젖힙니다. 이 같은 선결 처리 체계가 의미(means that) 부여하는 바인죽, 앞으로 우리가 맞닥뜨릴 (가령 잠시 뒤에 조만간 관찰 전개시켜 목격하게 될) 이른바 poly() 다항수식 전개항과 같은 작위에 의해 전개 창조되는 일체의 파생 발생 기인 스펙 특징 징후 변동물(derived features)들 역시도 결국에는 전부, 반복 재모집 창출 배포된 당해 신생 표본 풀 프레임 기단 위층에 몽땅 죄다 한 점 예외 없이 다시 재구성 탑재 장착 갱신되어 리-피팅(re-fit) 처리 동화 접수됨이 가능해진다는 사실에 다름 아닙니다.
