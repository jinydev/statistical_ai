5.1.5 Cross-Validation on Classification Problems

5.1.5 분류 문제에서의 교차 검증: 아날로그를 넘어 디지털 흑백 논리로!

In this chapter so far, we have illustrated the use of cross-validation in the regression setting where the outcome $Y$ is quantitative, and so have used MSE to quantify test error. But cross-validation can also be a very useful approach in the classification setting when $Y$ is qualitative. In this setting, cross-validation works just as described earlier in this chapter, except that rather than using MSE to quantify test error, we instead use the number of misclassified observations. For instance, in the classification setting, the LOOCV error rate takes the form 지금까지 우리가 다룬 교차 검증(CV) 시스템은 타겟 물건($Y$)이 키, 몸무게처럼 아날로그 선상의 연속된 숫자(정량적 quantitative) 인 ‘회귀(regression)’ 전장 위에서만 놀았습니다. 그래서 채점을 매길 때도 오답 거리의 덩치를 재는 오차 제곱합(MSE)을 무기로 썼죠. 자, 하지만 우리 CV 시스템은 “합격이냐 불합격이냐”, “개냐 고양이냐” 같은 디지털 흑백 카테고리 꼬리표(정성적 qualitative) 를 맞추는 ‘분류(classification)’ 전장에서도 미친 활약을 펼칠 수 있는 万能(만능) 치트키입니다. 룰은 예전 회귀판과 소름 돋게 똑같습니다(just as described). 단지 채점 방식만 스무스하게 달라지죠. 어설프게 점수 차이(MSE) 를 재는 거랑은 차원이 다르게, 여기선 얄짤없이 “틀리게 엉뚱한 이름표를 갖다 붙인 놈(오답 관측치)이 몇 명인가?” 를 빡세게 세어서 벌점을 때립니다. 예를 들어 이 살벌한 이분법적 분류 전장에서 LOOCV 의 에러 채점 성적표는 요런 형태로 씌어집니다.

\[\text{CV}_{(n)} = \frac{1}{n} \sum_{i=1}^n \text{Err}_i \quad (5.4)\]

where $\text{Err}_i = I(y_i \neq \hat{y}_i)$. The $k$-fold CV error rate and validation set error rates are defined analogously. 여기서 $\text{Err}_i = I(y_i \neq \hat{y}_i)$ 가 바로 가차 없는 ‘오답 판독기’ 입니다! 내 정답 $y_i$ 랑 니가 던진 예언 $\hat{y}_i$ 가 1밀리미터라도 다르면($\neq$) 얄짤없이 1점 깎이고(벌금 1), 똑같으면 0점(무죄) 처리되는 자비 없는 형틀이죠. $k$-폴드 CV 든 반반 무 많이 구식 검증이든 전부 다 이 무자비한 흑백 오답 판독기를 달고 똑같은 원리(analogously) 로 에러율을 산출해버립니다.

As an example, we fit various logistic regression models on the two-dimensional classification data displayed in Figure 2.13. In the top-left panel of Figure 5.7, the black solid line shows the estimated decision boundary resulting from fitting a standard logistic regression model to this data set. Since this is simulated data, we can compute the true test error rate, which takes a value of $0.201$ and so is substantially larger than the Bayes error rate of $0.133$. Clearly logistic regression does not have enough flexibility to model the Bayes decision boundary in this setting. We can easily extend logistic regression to obtain a non-linear decision boundary by using polynomial functions of the predictors, as we did in the regression setting in Section 3.3.2. Figure 5.7의 11시 방향 그림을 보세요. 시크한 검은 실선이 하나 쭉 그어져 있죠? 이건 튜닝 안 한 날 것 그대로의 기본형(standard) 뻣뻣한 로지스틱 기계가 자기 나름대로 심판의 영역선(결정 경계)을 찍찍 그어놓은 겁니다. 이건 신이 만든 시뮬레이션 가상 우주니까, 우린 전지적 작가 시점에서 ‘진짜 절대 오답률(True test error)’ 을 커닝할 수 있습니다. 계산해 보니 이 검은 선은 무려 $20.1\%$ ($0.201$) 의 헛스윙 타율을 기록 중이네요. 신이 내려준 이상적인 한계치인 베이즈(Bayes) 성적표 에러율이 고작 $13.3\%$ ($0.133$) 인 걸 감안하면 완전 똥볼(substantially larger) 을 차고 있습니다. 팩트는 명확합니다! 저 뻣뻣한 1차선 뼈대 가지곤, 신이 그린 화려한 베이즈 오아시스 경계선을 따라잡을 유연성(flexibility) 이 절망적으로 딸린다는 뜻입니다. 그럼 어떡합니까? 옛날 섹션 3.3.2에서 회귀선 좀 구부려보겠다고 수식에 마사지를 가했던 것처럼, 우리 로지스틱 대포에다가도 스펙 변수들을 제곱 세제곱(다항 함수) 뻥튀기하는 모터를 달아주면, 금세 고무줄처럼 춤추는 비선형(non-linear) 결정 경계를 공짜로 얻어냅니다. 힌트를 뽐볼 차원으로 끌어올린 이차(quadratic) 폭격 곡선 로지스틱의 간지나는 수식 폼은 대략 이렇습니다.

\[\log \left( \frac{p}{1-p} \right) = \beta_0 + \beta_1 X + \beta_2 X^2 \quad (5.5)\]

Figure 2.13의 가상 데이터 콜로세움에서 로지스틱 회귀 깡패들이 피팅 칼춤을 추는 현장. 아리따운 보라색 점선 실드가 바로 신계의 ‘베이즈 절대 방어선’. 시크한 검은 선들은 순서대로 우리의 1차, 2차, 3차, 4차원 로지스틱 용병들이 멋대로 그어버린 영토 국경선들입니다. 이 용병 4인방의 실전 수능 오답률 타율은 순서대로 20.1%, 19.7%, 16.0%, 그리고 16.2%를 치고 박고 있습니다. 참고로 신계 절대 한계치 모의고사 오답은 13.3%입니다.

The top-right panel of Figure 5.7 displays the resulting decision boundary, which is now curved. However, the test error rate has improved only slightly, to $0.197$. A much larger improvement is apparent in the bottom-left panel of Figure 5.7, in which we have fit a logistic regression model involving cubic polynomials of the predictors. Now the test error rate has decreased to $0.160$. Going to a quartic polynomial (bottom-right) slightly increases the test error. Figure 5.7의 1시 방향 방죽을 보세요. 아까 그 이차 곡선 모터를 타서 마침내 국경선이 뱀처럼 부드럽게 휘어졌습니다(curved). 그런데 솔직히 허세 떤 거에 비하면 실전 테스트 점수는 20.1% -> 19.7% 로 쬐~끔(only slightly) 줄고 말았네요 젠장. 하지만 진짜 떡상(apparent improvement) 은 7시 방향 방죽에서 터져 오릅니다! 이번엔 광기를 더해 3단 터보(3차 다항 cubic) 기어를 박아버린 미친 로지스틱 기계를 풀 가동시켰거든요. 와우! 실전 오답 스코어가 단숨에 16.0% ($0.160$) 로 수직 다이빙(decreased) 해버립니다. 신나서 마지막에 4단 터보(quartic) 뇌절을 시도한 5시 방죽을 보니, 오히려 점수가 16.2% 로 살짝 오르며(increases) 역효과 병크를 터뜨리는군요.

In practice, for real data, the Bayes decision boundary and the test error rates are unknown. So how might we decide between the four logistic regression models displayed in Figure 5.7? We can use cross-validation in order to make this decision. The left-hand panel of Figure 5.8 displays in black the 10-fold CV error rates that result from fitting ten logistic regression models to the data, using polynomial functions of the predictors up to tenth order. The true test errors are shown in brown, and the training errors are shown in blue. As we have seen previously, the training error tends to decrease as the flexibility of the fit increases. (The figure indicates that though the training error rate doesn’t quite decrease monotonically, it tends to decrease on the whole as the model complexity increases.) In contrast, the test error displays a characteristic U-shape. The 10-fold CV error rate provides a pretty good approximation to the test error rate. While it somewhat underestimates the error rate, it reaches a minimum when fourth-order polynomials are used, which is very close to the minimum of the test curve, which occurs when third-order polynomials are used. In fact, using fourth-order polynomials would likely lead to good test set performance, as the true test error rate is approximately the same for third, fourth, fifth, and sixth-order polynomials. 허나 진흙탕 실전판(In practice)에서 우리는 전지적 시점이 아니잖아요? 진짜 신계의 정답인 베이즈 선이나 내 진짜 실전 수능 오답률 봉투는 아무리 뒤져도 열리지 않는 블랙박스(unknown) 입니다. 그럼 씨가 마른 사막에서 방금 전 저 4개의 로지스틱 터보 엔진 중 대체 누굴 정식 스폰서로 채택해야 안 망할까요(how might we decide)? 이때 짜잔 하고 하늘에서 동아줄처럼 내려오는 구원자가 바로 교차 검증(cross-validation) 심판입니다! Figure 5.8 왼쪽 스테이지를 보세요. 시크하게 흐르는 검은 실선이 보이시죠? 우리가 엔진 기어를 무려 10단(10차 함수) 까지 무자비하게 올리면서 데이터에 10-폴드 훈련 뺑뺑이를 돌려 뽑아낸 ‘예언된 모의고사 에러 타율(CV 점수)’ 입니다. 그 뒤에 깔린 똥색 선은 우리가 이 악물고 모르척했던 절대 정답(True test error) 이고, 파란색 우울한 선은 속고만 사는 훈련장 방구석 점수(training error) 죠. 아까 닳도록 말했듯, 파란색 연습 점수는 우리 머신이 허리를 뱀처럼 이리저리 오버액션(유연성 증가) 할수록 땅바닥을 뚫고 끝없이 좋아 보이기만 합니다(물론 중간에 살짝 부들거리긴 해도 전체적으론 무조건 우하향하죠). 반면에, 실전 오답 스코어는 똥색 선처럼 전형적인 U자형 나락 롤러코스터 특성을 보여줍니다. 여기서 소름 돋는 포인트 투척! 우리의 검증 예언가(검은색 10-폴드 CV 궤도) 가 똥색 신계 정답 곡선의 형태를 제법 기가 막히게, 아주 존똑으로 베껴 그려내고(pretty good approximation) 쫓아가는 걸 볼 수 있습니다. 비록 높낮이 스펙 자체는 “야 너 꽤 잘 맞추네” 라며 오답 타율을 지레짐작 과소평가(underestimates) 치고 있긴 하지만요. 근데 튜닝 오타쿠들에겐 점수가 뭣이 중헙니까? 최저점(minimum) 타이밍이 생명이죠! 검은 CV 예언가는 기어가 4단 곡선(fourth-order)일 때 최저 오답 스코어 깃발을 꼽습니다. 이게 신계 정답 똥색 곡선이 3단(third-order) 근방에서 찍은 진짜 바닥 점 부근과 너무나도 숨 막히게 인접(very close to) 해 있습니다! 까놓고 말해서, 이 예언가의 말만 철석같이 믿고 4단 기어 엔진을 실전에 투입하더라도 대박 개꿀 성능을 터뜨렸을(likely lead to good) 겁니다. 왜냐면 똥색 정답선 바닥을 자세히 보면, 3단, 4단, 심지어 5단 6단까지 놈들의 오답 타율 바닥이 다 거기가 거기로 엇비슷한 평지 상태(approximately the same) 니까요!

The right-hand panel of Figure 5.8 displays the same three curves using the KNN approach for classification, as a function of the value of $K$ (which in this context indicates the number of neighbors used in the KNN classifier, rather than the number of CV folds used). Again the training error rate declines as the method becomes more flexible, and so we see that the training error rate cannot be used to select the optimal value for $K$. Though the cross-validation error curve slightly underestimates the test error rate, it takes on a minimum very close to the best value for $K$. 막간 타임! Figure 5.8 의 오른쪽 스테이지로 가볼까요? 이번엔 달리는 선수가 K-최근접 이웃 돌격대(KNN) 로 바뀌어, 쫄따구 패거리 인원수인 깡패 볼륨 $K$ (주의: 여기서 $K$는 파벌 쪼개기 CV용 10-폴드가 아니라, KNN 내 옆집 양아치 쪼무래기 숫자입니다!) 파워를 조작해 가며 궤도를 그립니다. 어휴 이것도 뻔해요. 기계가 지조 없이 방정맞아질수록(flexible, 여기서 $K$가 줄어들 때) 지 방구석 우물 안 개구리 같은 훈련장 점수(파란 선) 는 끝없이 좋아져 바닥을 기어갑니다. 그러니까 이 파란 사기꾼 점수판만 믿고선 최강의 투표 인원수 군단 $K$(optimal value) 를 절대 골라낼 수 없다는(cannot be used) 확신이 뼛속까지 박హి시죠?(we see that). 여기서도 우리의 구원자 등판! 예언가 CV 검은 선은 비록 진짜 똥색 정답보다 지레짐작 쪼그라들게(underestimates) 견적을 잘못 뽑고 징징거릴지언정(Though), 진짜 놈이 위치한 극강 효율의 $K$ 타이밍 최저점(best value for $K$) 주변을 뱀파이어처럼 찰거머리로 달라붙어(takes on a minimum very close to) 오답의 밑바닥 동굴을 기가 막히게 탐지해 냅니다!
