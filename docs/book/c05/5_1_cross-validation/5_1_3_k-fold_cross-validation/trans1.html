5.1.3 k-Fold Cross-Validation

5.1.3 k-폴드 교차 검증

An alternative to LOOCV is k-fold CV . This approach involves randomly dividing the set of observations into $k$ groups, or folds, of approximately equal size. The first fold is treated as a validation set, and the method is fit on the remaining $k - 1$ folds. The mean squared error, $\text{MSE}1$, is then computed on the observations in the held-out fold. This procedure is repeated $k$ times; each time, a different group of observations is treated as a validation set. This process results in $k$ estimates of the test error, $\text{MSE}_1, \text{MSE}_2, \dots, \text{MSE}_k$. The $k$-fold CV estimate is computed by averaging these values: LOOCV에 대한 대안(alternative)은 _k-폴드 CV 입니다. 이 접근법은 관측치 세트를 대략 동일한 크기의 $k$ 개 그룹, 즉 폴드(folds) 로 무작위 분리하는 것을 포함합니다. 첫 번째 폴드는 검증 세트로 취급(treated)되고, 해당 방법론은 나머지 $k - 1$ 개 폴드에 피팅됩니다. 그런 다음 제외된 폴드의 관측치들에 대해 평균 제곱 오차(mean squared error)인 $\text{MSE}_1$ 이 계산됩니다. 이 절차는 $k$ 번 반복되며, 매번 각기 다른 관측치 그룹이 검증 세트로 취급됩니다. 이 과정은 $k$ 개의 테스트 에러 추정치인 $\text{MSE}_1, \text{MSE}_2, \dots, \text{MSE}_k$ 를 낳습니다(results in). $k$-폴드 CV 추정치는 이 값들의 평균을 내어(averaging) 산출됩니다.

\[\text{CV}_{(k)} = \frac{1}{k} \sum_{i=1}^k \text{MSE}_i \quad (5.3)\]

Figure 5.5 illustrates the $k$-fold CV approach. Figure 5.5는 $k$-폴드 CV 접근법을 설명합니다.

$^1$ In the case of multiple linear regression, the leverage takes a slightly more complicated form than (3.37), but (5.2) still holds. $^1$ 다중 선형 회귀의 경우, 레버리지는 (3.37)보다 약간 더 복잡한 형태를 취하지만, (5.2)는 여전히 성립합니다. (역자 주: 이전 단원의 105페이지 주석(footnote)에서 이어짐)

Each of these fifths acts as a validation set (shown in beige), and the remainder as a training set (shown in blue). The test error is estimated by averaging the five resulting MSE estimates. FIGURE 5.5. 5-폴드 CV의 개략도(schematic) 표시. $n$ 개의 관측치 세트가 무작위로 서로 겹치지 않는(non-overlapping) 5개의 그룹으로 분리됩니다. 이들 각 1/5은 검증 세트(베이지색으로 표시) 역할을 하고, 나머지는 훈련 세트(파란색으로 표시) 역할을 합니다. 테스트 에러는 결과로 도출된 5개의 MSE 추정치의 평균을 내어 추정됩니다.

It is not hard to see that LOOCV is a special case of $k$-fold CV in which $k$ is set to equal $n$. In practice, one typically performs $k$-fold CV using $k = 5$ or $k = 10$. What is the advantage of using $k = 5$ or $k = 10$ rather than $k = n$? The most obvious advantage is computational. LOOCV requires fitting the statistical learning method $n$ times. This has the potential to be computationally expensive (except for linear models fit by least squares, in which case formula (5.2) can be used). But cross-validation is a very general approach that can be applied to almost any statistical learning method. Some statistical learning methods have computationally intensive fitting procedures, and so performing LOOCV may pose computational problems, especially if $n$ is extremely large. In contrast, performing 10-fold CV requires fitting the learning procedure only ten times, which may be much more feasible. As we see in Section 5.1.4, there also can be other non-computational advantages to performing 5-fold or 10-fold CV, which involve the bias-variance trade-off. $k$ 가 $n$ 과 같게 설정된 경우 LOOCV가 $k$-폴드 CV의 특수한 사례(special case)가 됨을 아는 것은 어렵지 않습니다. 실제(in practice) 적용에서는 주로 $k = 5$ 또는 $k = 10$ 을 사용하여 $k$-폴드 CV를 수행합니다. $k = n$ 보다 $k = 5$ 나 $k = 10$ 을 사용하는 이점은 무엇입니까? 가장 두드러진 이점은 연산적(computational)인 측면입니다. LOOCV는 통계적 학습 방법을 $n$ 번 피팅할 것을 요구합니다. 이는 연산적으로 비용이 많이 들(computationally expensive) 가능성이 있습니다(최소 제곱에 의해 피팅되는 선형 모델은 제외하며 이 경우에는 공식 (5.2)가 사용될 수 있음). 그러나 교차 검증은 거의 모든 통계적 학습 방법에 적용될 수 있는 매우 범용적인 접근법입니다. 일부 통계적 학습 방법은 계산 집약적(computationally intensive) 피팅 절차를 가지므로, LOOCV를 수행하는 것은 특히 $n$ 이 엄청나게 클 경우 연산 문제를 야기할 수 있습니다. 대조적으로, 10-폴드 CV를 수행하는 것은 훈련 절차를 고작 열 번만 피팅할 것을 요구하므로 훨씬 실행 가능(feasible)할 수 있습니다. 우리가 Section 5.1.4에서 보게 되듯, 편향-분산 트레이드오프(bias-variance trade-off)를 포함하는 5-폴드 또는 10-폴드 CV 수행에는 다른 비연산적 이점 또한 있을 수 있습니다.

The right-hand panel of Figure 5.4 displays nine different 10-fold CV estimates for the Auto data set, each resulting from a different random split of the observations into ten folds. As we can see from the figure, there is some variability in the CV estimates as a result of the variability in how the observations are divided into ten folds. But this variability is typically much lower than the variability in the test error estimates that results from the validation set approach (right-hand panel of Figure 5.2). Figure 5.4의 우측 패널은 Auto 데이터 세트에 대한 서로 다른 9가지의 10-폴드 CV 추정치를 표시하며, 이들 각각은 관측치들을 무작위로 10번씩 각기 다르게 분할한 결과입니다. 그림에서 볼 수 있듯이, 관측치가 10개의 폴드로 분할되는 방식의 가변성으로 인한 약간의 변동(some variability)이 CV 추정치 내에 존재합니다. 그러나 이 가변성은 전형적으로(typically) 검증 세트 접근 방식(Figure 5.2 우측 패널)에서 초래되는 테스트 에러 추정치의 가변성보다 훨씬 더 낮습니다.

When we examine real data, we do not know the true test MSE, and so it is difficult to determine the accuracy of the cross-validation estimate. However, if we examine simulated data, then we can compute the true test MSE, and can thereby evaluate the accuracy of our cross-validation results. In Figure 5.6, we plot the cross-validation estimates and true test error rates that result from applying smoothing splines to the simulated data sets illustrated in Figures 2.9–2.11 of Chapter 2. The true test MSE is displayed in blue. The black dashed and orange solid lines respectively show the estimated LOOCV and 10-fold CV estimates. In all three plots, the two cross-validation estimates are very similar. In the right-hand panel of Figure 5.6, the true test MSE and the cross-validation curves are almost identical. In the center panel of Figure 5.6, the two sets of curves are similar at the lower degrees of flexibility, while the CV curves overestimate the test set MSE for higher degrees of flexibility. In the left-hand panel of Figure 5.6, the CV curves have the correct general shape, but they underestimate the true test MSE. 우리가 현실 데이터를 검증(examine)할 때 우리는 진짜(true) 테스트 MSE를 알 수 없으며, 따라서 교차 검증 추정치의 정확도를 결정하기가 어렵습니다. 그러나 우리가 시뮬레이션 된 데이터를 검토한다면 진짜 테스트 MSE를 산출할 수 있고, 이를 통해 교차 검증 결과의 정확도를 평가할 수 있습니다. Figure 5.6에서, 우리는 Chapter 2의 Figures 2.9–2.11에서 도해된(illustrated) 시뮬레이션 데이터 세트에 스무딩 스플라인(smoothing splines)을 적용한 결과로 나타나는 교차 검증 견적들과 진짜 테스트 에러율을 도표로 작성합니다. 진짜 테스트 MSE는 파란색으로 표시됩니다. 검은색 점선(dashed)과 주황색 실선(solid)은 각각(respectively) LOOCV 및 10-폴드 CV의 추정치를 모여줍니다. 세 개의 플롯(plots) 모두에서 두 교차 검증 추정치는 무척 유사합니다. Figure 5.6의 오늘쪽 패널에서 진짜 테스트 MSE와 교차 검증 곡선은 거의 동일합니다. Figure 5.6의 중앙 패널에서 두 세트의 곡선들은 자유도(degrees of flexibility)가 낮을 때 유사한 반면, 더 높은 자유도에 대해서는 CV 곡선이 테스트 세트 MSE를 과대평가(overestimate)합니다. Figure 5.6의 좌측 패널에서 CV 곡선들은 올바른 일반 형태(correct general shape)를 지니고 있으나, 진짜 테스트 MSE를 과소평가(underestimate)합니다.

FIGURE 5.6. True and estimated test MSE for the simulated data sets in Figures 2.9 ( left ), 2.10 ( center ), and 2.11 ( right ). The true test MSE is shown in blue, the LOOCV estimate is shown as a black dashed line, and the 10 -fold CV estimate is shown in orange. The crosses indicate the minimum of each of the MSE curves. FIGURE 5.6. Figures 2.9 (좌), 2.10 (중), 그리고 2.11 (우)의 시뮬레이션 데이터 세트들에 대한 진짜(True) 및 추정된 테스트 MSE. 진짜 테스트 MSE는 파란색, LOOCV 추정치는 검정 점선으로, 10-폴드 CV 견적치는 주황색으로 표시됩니다. 십자가 기호는 각각 MSE 곡선의 최솟값을 나타냅니다.

When we perform cross-validation, our goal might be to determine how well a given statistical learning procedure can be expected to perform on independent data; in this case, the actual estimate of the test MSE is of interest. But at other times we are interested only in the location of the minimum point in the estimated test MSE curve. This is because we might be performing cross-validation on a number of statistical learning methods, or on a single method using different levels of flexibility, in order to identify the method that results in the lowest test error. For this purpose, the location of the minimum point in the estimated test MSE curve is important, but the actual value of the estimated test MSE is not. We find in Figure 5.6 that despite the fact that they sometimes underestimate the true test MSE, all of the CV curves come close to identifying the correct level of flexibility—that is, the flexibility level corresponding to the smallest test MSE. 교차 검증을 수행할 때, 우리의 목표는 독립적인 데이터에서 특정 통계적 학습 절차가 얼마나 잘 작동할 것으로 기대되는지 파악하는 것일 수 있습니다. 이 경우, 실제 테스트 MSE 추정치가 우리의 관심사(of interest)일 것입니다. 그러나 다른 때 우리는 오직 추정된 테스트 MSE 곡선 상의 최솟값 위치(location of the minimum point) 에만 관심이 있습니다. 왜냐하면 우리가 가장 낮은 테스트 에러를 유발하는 방법을 가려내기 위해(identify) 수많은 통계 학습 방론론에 혹은 여러 다른 유연성 척도를 사용하는 단일 방법에 교차 검증을 수행 중일 수 있기 때문입니다. 이러한 목적(purpose)을 위해, 추정된 테스트 MSE 곡선의 진짜 값 (actual value) 자체가 아니라 최솟값을 지닌 지점의 위치가 무엇보다 중요합니다. 우리는 Figure 5.6에서 그들이 종종 진짜 테스트 MSE를 과소평가한다는 사실에도 불구하고(despite the fact that), 모든 CV 곡선들이 정확한 유연성의 척도—즉 가장 작은 테스트 MSE에 상응하는(corresponding to) 유연성 레벨—를 가려내는 데 무척 근접해 있음(come close to)을 발견합니다.
