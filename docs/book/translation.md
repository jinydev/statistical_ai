# 통계적 학습 (Statistical Learning) 번역 가이드 (통번역 스타일)

이 가이드는 `An Introduction to Statistical Learning`의 마크다운 번역 시 유지해야 하는 1:1 교차 번역(Alternating Bilingual) 스타일 규칙을 정의합니다. `c01`의 번역 스타일을 기준으로 작성되었습니다.

## 0. 직역 원칙 (Literal Translation) - 추가됨
- **원문 충실성**: 영어 원문의 의미를 자의적으로 해석하여 부연 설명을 덧붙이거나 의역(paraphrase)하지 마십시오.
- **최대한 직역**: 원문의 구조와 단어를 최대한 직역 수준으로 번역하여 간결하고 명확하게 유지합니다. 원본 텍스트에 없는 부가적인 설명이나 생각은 절대 추가하지 않습니다.

## 1. 문장 단위 분리 (Sentence Segmentation)
- **마침표 기준 분리**: 원문 영어 마크다운 텍스트를 마침표(`.`), 물음표(`?`), 느낌표(`!`) 등을 기준으로 하나의 문장 단위로 분리하여 각 줄에 배치합니다. (단, Mr. 이나 e.g. 와 같은 약어의 마침표는 문장 끝으로 간주하지 않습니다.)
- 단락(Paragraph) 단위로 뭉쳐있는 영어 원문을 반드시 문장 단위로 쪼개야 합니다.

## 2. 1:1 교차 번역 (Alternating Bilingual Translation)
- **포맷**: 한 줄의 영어 원문 문장 아래에 빈 줄(Blank line)을 하나 두고, 그 다음 줄에 해당 문장의 한글 번역본을 작성합니다. 한글 번역본 다음에도 빈 줄을 하나 둡니다.
- **구조 예시**:
  ```markdown
  These tools can be classified as *supervised* or *unsupervised*.

  이러한 도구들은 *지도 학습* 또는 *비지도 학습*으로 분류될 수 있습니다.
  ```

## 3. 제목 및 헤더 (Headers)
- 문장과 동일하게 영어 원문 헤더 작성 후 빈 줄을 띄우고 한글 번역 헤더를 작성합니다. 헤더 레벨(`#`, `##` 등)은 동일하게 유지해야 합니다.
- **예시**:
  ```markdown
  ## Wage Data

  ## 임금 데이터
  ```

## 4. 마크다운 문법 보존 (Preserving Markdown Syntax)
- **서식 유지**: 굵게(`**bold**`), 기울임(`*italic*` 등), 인라인 코드(`` `code` ``), LaTeX 수식 표기(`$math$`, `$$math$$`) 등은 원문과 동일하게 한글 번역문에서도 보존해야 합니다.
- **기술 용어 표기**: 중요한 변수명이나 데이터셋 이름, 또는 명확한 전달이 필요한 용어의 경우 인라인 코드 또는 원어를 병기합니다. (예: `` `wage(임금)` ``)

## 5. 이미지 및 캡션 (Images & Captions)
- **이미지 태그**: 이미지를 삽입하는 마크다운(`![alt](./img/...)`)은 영어/한글 교차 없이 한 번만 삽입합니다. (원본 유지)
- **캡션 번역**: 이미지 하단에 굵은 글씨(`**FIGURE 1.1.**`)와 이탤릭체로 구성되는 캡션 텍스트는 본문과 동일하게 영어 캡션 작성 후 한글 캡션으로 번역하여 배치합니다.

## 6. 프론트매터 및 하위 목차 (Frontmatter & Sub-Chapters)
- 페이지 상단의 `---` 로 둘러싸인 YAML 프론트매터(Frontmatter)는 번역하지 않고 원본 그대로 유지합니다.
- 문서 하단에 추가된 `## Sub-Chapters (하위 목차)` 부분은 기존에 작성된 한글 내용을 유지하며 별도로 영어 역번역을 하지 않습니다.
