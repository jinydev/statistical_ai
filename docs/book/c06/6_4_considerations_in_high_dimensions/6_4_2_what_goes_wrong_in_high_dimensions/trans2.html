6.4.2 고차원에서 발생하는 문제점 (What Goes Wrong in High Dimensions?)

In order to illustrate the need for extra care and specialized techniques for regression and classification when $p > n$, we begin by examining what can go wrong if we apply a statistical technique not intended for the high-dimensional setting. For this purpose, we examine least squares regression. But the same concepts apply to logistic regression, linear discriminant analysis, and other classical statistical approaches. 피처 특성의 수가 너무 비대해져 $p > n$ 환경이 초래될 때, 회귀나 분류 모델 설계에 있어 왜 그토록 눈물을 참아가며 각별한 주의를 기울여야 하고 거기에 특별히 고안된 테크닉 기법들이 도입되어야만 하는지 명확히 설명해 드리고자 합니다. 이를 뒷받침하기 위해 앞서 배운 일반 저차원 통계 기법 기술을 아무런 조치 없이 고차원 데이터 세팅에 무작정 때려 박았을 때 과연 어떤 치명적인 오작동 사태가 초래될 수 있는지부터 낱낱이 파헤쳐보겠습니다. 이 검증 목적을 실험하기 위해 만만한 기본 모델인 최소 제곱 회귀 단면을 집중적으로 끌고 와서 살펴볼 것입니다. 허나 사실 여기서 우리가 얻게 도출될 통계의 참담한 기본 개념적 맹점들은 로지스틱 회귀나 선형 판별 분석 같은 모델뿐만 아니라, 기타 모든 낡고 오래된 구닥다리 고전적 통계 접근 평가 방식들 전역 무대에 사실 동일하게 적용되어 똑같이 무너집니다.

When the number of features $p$ is as large as, or larger than, the number of observations $n$, least squares as described in Chapter 3 cannot (or rather, should not) be performed. The reason is simple: regardless of whether or not there truly is a relationship between the features and the response, least squares will yield a set of coefficient estimates that result in a perfect fit to the data, such that the residuals are zero. 아시다시피 특성 변수 객체들의 총 수량 $p$가, 확보된 데이터 행인 관측치 개수 $n$과 동급으로 몸집이 거대해지거나 심지어 $n$보다 단위가 더 커져 넘어버리는 경우라면, 3장에서 우리가 신나게 배운 기가 막힌 방식인 오리지널 최소 제곱법 기단은 결코 정상적으로 수행될 수가 없습니다 (아니 정확하게 지적하자면, 절대 함부로 수행해서는 안 통제됩니다(should not)). 그 파멸의 이유는 아주 심플합니다. 구비된 특성 요소들과 목표 지향 타겟인 종점 반응 변수들 사이에 진짜배기 그 속내에 가치 있는 진정한 연관성 관계가 실제로 존재하든 안 하든 그 실체와는 아무런 상관없이, 최소 제곱법은 눈가림식으로 단지 당장의 수치 잔차(residuals) 찌꺼기 추정차가 모조리 흔적 없이 0으로 증발해버리도록 아주 계산적으로 데이터를 무단 완벽하게 가짜 적합시켜 계산 맞춰버리는 계수 추정치 결과물들을 시스템적으로 강제 통제 지시해 토해내 버릴 것이기 때문입니다.

An example is shown in Figure 6.22 with $p = 1$ feature (plus an intercept) in two cases: when there are 20 observations, and when there are only two observations. When there are 20 observations, $n > p$ and the least squares regression line does not perfectly fit the data; instead, the regression line seeks to approximate the 20 observations as well as possible. On the other hand, when there are only two observations, then regardless of the values of those observations, the regression line will fit the data exactly. This is problematic because this perfect fit will almost certainly lead to overfitting of the data. In other words, though it is possible to perfectly fit the training data in the high-dimensional setting, the resulting linear model will perform extremely poorly on an independent test set, and therefore does not constitute a useful model. In fact, we can see that this happened in Figure 6.22: the least squares line obtained in the right-hand panel will perform very poorly on a test set comprised of the observations in the left-hand panel. The problem is simple: when $p > n$ or $p \approx n$, a simple least squares regression line is too flexible and hence overfits the data. 우리는 그림 6.22를 통해 좋은 예시를 봅니다. 해당 도식은 단 하나의 예측 특성 피처 $p = 1$을 가진 모델을 두 가지 관측 빈도 조건(관측치가 20개일 때와 2개일 때)으로 나누어 보여줍니다. 20개 관측치 세팅에서는 $n > p$ 기본 조건이 성립하여 최소 제곱 회귀선이 데이터를 오차 없이 완벽히 관통하진 않습니다. 대신 회귀선은 20개 관측점을 최대한 근사하려 노력합니다. 반면 관측치가 단 2개뿐이라면 상황이 달라집니다. 단 둘 뿐인 관측값의 크기와는 무관하게 두 점을 이은 선형 모델이 데이터를 오차 없이 정확하게 적합시킵니다. 데이터에 대한 이 완벽한 적합(perfect fit)은 거의 예외 없이 모델의 과적합(overfitting)으로 이어지므로 아주 치명적인 문제입니다. 즉, 고차원 환경에서는 주어진 훈련 데이터를 오차 없이 완벽하게 적합시키는 게 기능적으로 통계 조작상 가능할진 몰라도, 그렇게 돌출된 선형 모델은 독립적인 테스트 평가 세트에서는 그야말로 형편없는 최악의 성과를 기록해 결국 쓸모없는 모델이 됩니다. 실제로 이 비극은 그림 6.22에서 동일하게 벌어집니다. 오른쪽 도식 패널에서 오차 없이 얻어진 최소 제곱선은 왼쪽 패널의 별도 독립 관측치로 이뤄진 테스트 세트 구동 환경에서는 매우 밑바닥 수준의 성능을 보일 것입니다. 결론은 이렇습니다: $p > n$ 이거나 $p \approx n$ 의 체제일 때 최소 제곱 회귀모델은 예측선을 과도하게 유연(flexible) 해지도록 방치하므로 훈련 데이터를 극심하게 과대 적합(overfitting)시켜 버립니다. “””

Figure 6.23 further illustrates the risk of carelessly applying least squares when the number of features $p$ is large. Data were simulated with $n = 20$ observations, and regression was performed with between 1 and 20 features, each of which was completely unrelated to the response. As shown in the figure, the model $R^2$ increases to 1 as the number of features included in the model increases, and correspondingly the training set $\text{MSE}$ decreases to 0 as the number of features increases, even though the features are completely unrelated to the response. On the other hand, the $\text{MSE}$ on an independent test set becomes extremely large as the number of features included in the model increases, because including the additional predictors leads to a vast increase in the variance of the coefficient estimates. Looking at the test set MSE, it is clear that the best model contains at most a few variables. However, someone who carelessly examines only the $R^2$ or the training set $\text{MSE}$ might erroneously conclude that the model with the greatest number of variables is best. This indicates the importance of applying extra care when analyzing data sets with a large number of variables, and of always evaluating model performance on an independent test set. 그림 6.23 도면은 통계 투입 특성의 개수인 $p$가 클 때 이렇게 부주의하게 막무가내로 최소 제곱 기법을 함부로 적용하는 행위의 치명적 위험성을 훨씬 더 적나라하게 경고해 묘사합니다. 총 $n = 20$개의 단단한 관측치 객체로 가상 시뮬레이션 합성된 데이터상에, 1개부터 최대 20개 사이의 다중 피처 특성을 집어넣어 회귀 추론을 수행해 봅니다. 이때 투입시킨 모든 예측 특성 변수들은 애당초 목표 반응 타겟 변수 방향과는 정말 눈곱만큼도 관계없이 제멋대로 노는 완전히 독립적 무작위 가짜 값들이었습니다. 그런데 그림 산출 뷰에서 알 수 있듯, 모델 훈련망에 투입 포함시키는 가짜 특성 개수가 점진적으로 하나둘 늘어날수록 마법처럼 훈련 세트 $R^2$ 점수는 비상식적으로 거짓말처럼 1을 향해 뚫고 치솟으며, 동시에 훈련 모델 세트 측 평가 $\text{MSE}$ 지수는 꿈속의 오차율 제로인 0을 향해 점진 감소해 수렴합니다. 심지어 모델에 밀어 넣은 예측 특성 피처들이 실제 판별 종점 반응 점수엔 완전히 일말의 상관도 없고 일치 안 하는 무의미한 독립 요소임에도 불구하고 일어나는(even though the features are completely unrelated to the response) 파멸적 수리적 통계 마술 착시 눈속임 현상입니다. 하지만 이 동전 뒷면으로 역전해 단면 독립적인 공간으로 격리된 새로운 별개 자원 무대인 독립 분할 테스트 검증 세트(independent test set) 진영 내의 검증 $\text{MSE}$ 수치는 훈련 통계와는 다르게 모형 변수의 수가 추가될수록 우주 폭발하듯 극렬 거대하게 터져 치솟게 됩니다. 쓸모없는 가변성 단면 부가 예측 변수를 닥치는 대로 닥닥 긁어 모형에 꾸역 끼워 넣고 투입을 남발하는 행위는 결국 측정 추정 계수 변동 폭, 즉 어마무시한 통제 불능 분산(variance)의 극한 변동 증폭 확산을 막대하게 자극해버리기 때문입니다. 평가 잔차 테스트 측 오차 계산 통계 MSE를 냉정하게 객관적으로만 살피면, 기껏해야 아주 극소수 소수의 통제 변수만을 남겨 차용한 한정 모델 구조가 명백하게 가장 빛나는 최고의 최적 성능을 찍는 베스트 모델임이 자명해집니다. 하지만 이러한 고차원 데이터의 무서운 속내 모순을 간과한 채, 섣불리 단순하게 눈가림 척도인 모의 $R^2$ 지표나 당장 눈앞 모델 측정 진영의 편향적 훈련 오류 모의 $\text{MSE}$ 평가 수치만 얕게 치중해 살피는 사람은 변수를 최대한 많이 무식하게 끌어모은 과적합 돼지 피처 모델이 가당찮게 가장 최고라는 최악의 잘못된 헛발질 착각 결론에 빠지고 말 것입니다. 이 사태는 막대한 수치 단위 예측 배열 변수들을 보유한 거대 관측 고차원 데이터 진영 분석 무대에서 절대적 경각심 주의 통제 방식 잣대를 강제 적용해야 함과 동시에, 항상 무조건 최종 판단 모델 도출 성능 평가는 철저 완벽하게 격리 확보 통제된 분리 궤도 상 외부 척도의 독립 단면 테스트 세트 무대 상에서 엄격히 치러 단행 판별하는 습관이 목숨만큼 중요하다는 것을 적나라하게 가르쳐 줍니다.

In Section 6.1.3, we saw a number of approaches for adjusting the training set RSS or $R^2$ in order to account for the number of variables used to fit a least squares model. Unfortunately, the $C_p$, AIC, and BIC approaches are not appropriate in the high-dimensional setting, because estimating $\hat{\sigma}^2$ is problematic. (For instance, the formula for $\hat{\sigma}^2$ from Chapter 3 yields an estimate $\hat{\sigma}^2 = 0$ in this setting.) Similarly, problems arise in the application of adjusted $R^2$ in the high-dimensional setting, since one can easily obtain a model with an adjusted $R^2$ value of 1. Clearly, alternative approaches that are better-suited to the high-dimensional setting are required. 앞선 6.1.3절 파트에서 우리는 최소 제곱 모델 추정을 돕기 위해 훈련 세트 오차율 규격 단위 $\text{RSS}$ 통계나 혹은 모형 척도 조율 $R^2$ 지표 등을 반영 조치하여 보정해 주는 통제 대안들을 여럿 다르게 파헤쳐 살폈습니다. 너무나도 애석하게도 이런 대안들로 꼽혔던 유명한 $C_p$, $\text{AIC}$, 그리고 $\text{BIC}$ 기법 추론 수단들은 고차원 예측 모델 세팅 환경에서는 절대 맞물리지 못하며 부적합합니다. 왜냐하면 고차원에선 분산 추정치 지수 $\hat{\sigma}^2$를 도출 단면 계산해 내는 척도 행위 자체가 근본적으로 심각한 결함 모순 문제가 파생되기 때문입니다. (가령, 3장 모의 과정에서 다룬 $\hat{\sigma}^2$ 기본 연산 추론 공식을 꽉 막힌 이곳 고차원 시스템 세팅 환경 궤도에 대입해 돌려버리면 그 추정 결괏값이 무조건 공식 산출 $\hat{\sigma}^2 = 0$ 으로 텅 빈 깡통 도출 붕괴되어 망가집니다.) 이와 동일한 궤도의 원리로, 편법적인 조정된 모의 $R^2$ 구도를 고차원 환경에 억지 적용할 때도 똑같은 부수 문제점들이 도출됩니다. 왜냐하면 이렇게 관측치 한계를 뛰어넘는 무수한 피처 차원 틈에선, 맘만 먹으면 쓸데없는 변수를 무작정 집어넣어 구색 명분으로 누구든 너무 쉽고 간단하게 거짓 보정된 $R^2$ 평가 무결 최종 지수값이 1이라는 사기 완벽 모델 기적 달성을 이뤄낼 수 있기 때문입니다. 결론적으로, 이렇게 특수하게 한쪽 치수로 뚱뚱하게 기우는 비대한 고차원 세팅 환경 분석 무대에 걸맞게 훨씬 더 유기적으로 부합하는 대안적 신규 모델 파생 구조 접근 해결책 기조들이 앞으로의 통계 진영 모델 미래엔 절실히 필연적으로 요구 기원됩니다.
