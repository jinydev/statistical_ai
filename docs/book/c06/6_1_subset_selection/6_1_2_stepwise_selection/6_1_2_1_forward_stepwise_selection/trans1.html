Forward Stepwise Selection

전진 단계 선택법 (Forward Stepwise Selection)

Forward stepwise selection is a computationally efficient alternative to best subset selection. While the best subset selection procedure considers all $2^p$ possible models containing subsets of the $p$ predictors, forward stepwise considers a much smaller set of models. Forward stepwise selection begins with a model containing no predictors, and then adds predictors to the model, one-at-a-time, until all of the predictors are in the model. In particular, at each step the variable that gives the greatest additional improvement to the fit is added to the model. More formally, the forward stepwise selection procedure is given in Algorithm 6.2. ‘전진 단계 선택법(Forward stepwise selection)’ 은 앞선 최적 부분집합 선택 시스템 구조의 방만한 단점을 극복하는, 계산 측면에서 훨씬 가볍고 지극히 연산 효율적인(computationally efficient) 핵심 우회 대안(alternative) 기법입니다. 최적 부분집합 선택 방식이 $p$ 개 예측 변수 구성으로 자아낼 수 있는 모든 $2^p$ 가짓수의 모델 표본 파편군 전체(all $2^p$ possible models)를 몽땅 다 무식하게 검열 수색하는 것과 대조되게, 전진 단계법(forward stepwise)은 이보다 훨씬 확연하게 적은 초소형 규모의 기동 탐색 모델 경우의 수만을 고려하여 전개됩니다. 전진 단계 선택법은 일차적으로 내부에 단 하나의 예측 변수도 담지 않은 완전한 깡통 영(null) 모델에서 생성을 단출하게 출발 시작(begins with) 하며, 이후 그 빈 모델 진영 내부로 예측 변수들을 한 턴에 오직 단 하나씩만(one-at-a-time) 점진 도입 투입 추가 융합하는 상승 기동 방식을, 최종적으로 주어졌던 예측 변수 전원이 모조리 다 모델에 포함 탑재 포용 수렴될 때까지 순차 차례로 추가 확장해 나아갑니다. 특정지어 말하자면, 매번 이루어지는 각 상승 등판 스텝 추가 단계(at each step) 국면마다, 당대 모델 내부의 피팅 역량 및 예측 성능 타협 스코어에 가장 최대의 획기적이고 긍정적인 부가 상승폭 향상치(the greatest additional improvement)를 선사 기여 조력 부여해 주는 1순위 일등 변수 성분 하나만이 발탁 지목 간택되어 모델 내부로 최종 추가 투입 흡수(added to) 됩니다. 이를 정석 포맷을 띠어 보다 격식 있고 정제된 수리적 척도 논리로 표현 서술하자면, 후속 ‘알고리즘 6.2’ 에 적시 서술된 절차 공식 체계와 완벽 일치 부합(given in) 합니다.

Unlike best subset selection, which involved fitting $2^p$ models, forward stepwise selection involves fitting one null model, along with $p − k$ models in the $k$th iteration, for $k = 0, \dots, p − 1$. This amounts to a total of $1 + \sum_{k=0}^{p-1} (p - k) = 1 + p(p+1)/2$ models. This is a substantial difference: when $p = 20$, best subset selection requires fitting 1,048,576 models, whereas forward stepwise selection requires fitting only 211 models.[^2] 도합 무려 $2^p$ 개라는 막대한 파생물 모델 객체들을 강제로 일일이 피팅(fitting) 적합 시험대에 올려 시달리게 만들었던 최적 부분집합 방식과는 전혀 상이한 궤도로, 현 단계 전진 선택 방식 체제는 우선 단 1개의 최초 영 모델 객체를 피팅하는 수고를 시작으로 곁들여 수반하며, 그 주변으로 매번 거치는 $k$ 번째 반복 회전 루프 단계($k$th iteration, 단 범위는 $k = 0, \dots, p − 1$ 설정) 진영 내에서는 잔류한 미개척 변수들의 수에 기인한 도합 단지 $p - k$ 개짜리 각 층위별 후보 모델 세트들만을 시험 심사 적합 시키는 여유로운 과정을 거칩니다. 이를 전체 범위 단위로 누적 환산 집계 결산(amounts to) 해보면, 오직 $1 + \sum_{k=0}^{p-1} (p - k) = 1 + p(p+1)/2$ 개수에 해달하는 규모 모델들만이 전역 생성 도출된다는 경제적인 물리 계산 사실입니다. 이 현상은 대단히 압도적이고 극명 확연한 근본적 수치 격차 효용 극대 이면을(substantial difference) 보여줍니다: 일례로 초기 투입 고려 변수 $p = 20$ 개라는 상황이 주어진다면, 최적 부분집합 모델법은 자그마치 자력으로 미련하게 1,048,576개의 방대한 모델 세트를 형성 조립 적합 시켜 돌려야 하지만 반면에, 현 단계 전진 전개법은 고작해야 귀엽게 211개에 불과한 소형 스펙 모델 풀 만을 대상으로 피팅 조작을 치르면 족하기 때문입니다.

[^2] 비록 전진 단계 선택법이 겉으로는 표면상 $1 + p(p+1)/2$ 개의 모델군 경우의 수만 통과 고려 판단하는 수치를 띠긴 하지만, 이 체제 구조는 거대한 모델 탐사 영토 무대 상에서 이미 방향성이 설정 유도 통제된 유효 가이드 추적 궤도 동선 조종 심사 지휘 심사망(a guided search)의 탐사 프로세스를 전격 구축 발휘 띄고 있기에, 실질적 기저 내부 수리적 연산 검열로 치환 이면 판별 통과 심사(considered)된 거대 포괄 실질 모델 공간(effective model space)은 막강하게도 앞서 언급된 $1 + p(p+1)/2$ 개수의 단층 스펙보다 훨씬 더 광대 방대 우세 상회 포괄 수립하는 통계적 실질 스펙트럼(substantially more than)을 자랑 내재 내포 포괄 구축하게 됩니다.

In Step 2(b) of Algorithm 6.2, we must identify the best model from among those $p−k$ that augment $M_k$ with one additional predictor. We can do this by simply choosing the model with the lowest RSS or the highest $R^2$. However, in Step 3, we must identify the best model among a set of models with different numbers of variables. This is more challenging, and is discussed in Section 6.1.3. 앞서 기술한 알고리즘 6.2 순차의 Step 2(b) 구간 단계에서, 우리들은 기저 베이스캠프 $M_k$ 뼈대에다 1개 여분의 신입 예측 변수를 차출 보강 추가 이입(augment) 시켜 탄생한, 그 당대 동일 체급 부피 사이즈의 $p - k$ 개 동기 분파 경쟁 모델 진영 풀 상에서 결국 가장 단연 출중한 단 하나만의 에이스 모델을 반드시 추려내 1등 발탁 간택 식별(identify) 조치해야만 합니다. 이는 그저 통상 단순 가장 최소 바닥 지수 RSS 오차를 내거나 아니면 제일 극단의 최고봉 $R^2$ 평가 지표(highest) 를 뽑아낸 객관 수치 최고 선방 모델을 단순히 채택 지정(simply choosing) 해냄으로써 가볍게 성취 달성(do this) 이뤄낼 수 있습니다. 허나 이후 난제로 맞이하게 될 마지막 Step 3 단계 구간 상황에서는 전혀 성질이 다른, 무려 예측 변수 개수(numbers of variables) 부피가 저마다 각기 판이 이질 상이한(different) 이종 체급 다수 모델 연합 풀(a set of models) 속에서 절대 에이스 1등 모델 객체를 통과 식별 색출 지목(identify) 해 내야만 할 커다란 과제에 처합니다. 이는 앞선 동일 체급 간의 비교 판별 과정 난이도보다 훨씬 아찔 복잡 난해 벅찬 통과(more challenging) 어려움 고비에 봉착하게 되며, 이에 관련된 상세 우회 돌파 대응 구사 해법 논의 서술(discussed in) 관련 부연 언급은 곧바로 다음 섹션 6.1.3 단락(Section 6.1.3) 무대 지점에서 면밀 집중 이어 진행 다룰(discussed) 예정입니다.

Forward stepwise selection’s computational advantage over best subset selection is clear. Though forward stepwise tends to do well in practice, it is not guaranteed to find the best possible model out of all $2^p$ models containing subsets of the $p$ predictors. For instance, suppose that in a given data set with $p = 3$ predictors, the best possible one-variable model contains $X_1$, and the best possible two-variable model instead contains $X_2$ and $X_3$. Then forward stepwise selection will fail to select the best possible two-variable model, because $M_1$ will contain $X_1$, so $M_2$ must also contain $X_1$ together with one additional variable. 전진 단계 알고리즘(Forward stepwise)은 계산적 이점(computational advantage)이 무식한 최적 부분집합 방식(best subset selection)에 비해 투명하게 압도적(clear)이라는 무기가 있습니다. 이 전진 단계 방식이 현실 필드 현장(in practice)에서 꽤나 제 몫을 하며 수려한 실적 퍼포먼스를 달성하는 기특한 경향(tends to do well)이 여실히 증명되긴 합니다만(Though), 그렇다 한들 이 녀석이 저 거대한 $2^p$ 개의 전체 부분집합 모델들 우주(out of all $2^p$ models) 안에서 기어코 진짜 베기 유일무이 1등 절대 우수 모델(the best possible model)을 정확히 찍어 찾아낼(find) 것이라는 어떤 무결한 보장 입증 단언(guaranteed) 같은 건 절대 존재하지 않습니다(not). 치명적인 함정 일례(For instance)를 하나 들어보겠습니다. 만약 투입 예측 변수가 $p = 3$ 명으로 구성된 작달막한 데이터 세트가 있는데, 운명의 장난처럼 오직 단일 변수 1개만 쓰는 체급에서의 세상 가장 우수한 모델(best possible one-variable model)은 $X_1$ 녀석을 장착(contains) 했고, 반면에 변수를 2개나 쓴 두 번째 체급에서의 진짜배기 가장 우수 모델(best possible two-variable model)은 아까의 $X_1$ 따위는 버리고 완전히 다른 $X_2$ 와 $X_3$ 녀석들을 파트너로 편입 장착(contains instead) 한 돌발 상황(suppose that)이 발생했다고 상상해 봅시다. 그렇다면(Then) 우리의 순진무구한 이 ‘전진 단계 선택(forward stepwise selection)’ 알고리즘 방조 방식은, 저 기막힌 ‘진짜 우수 2-변수 모델’을 발굴 지목 섭렵(select) 하는 데에 완전 처참하게 길을 잃고 실패(fail to) 해버리고 맙니다! 도대체 왜냐고요?(because) 제일 첫 단계인 미니 베이스캠프 $M_1$ 진영을 꾸릴 때 이 알고리즘은 무조건 당시 1등이었던 $X_1$ 요원 한 명만 편애해 고집스레 포용 탑재(contain $X_1$) 시키도록 설계되었고, 그 여파로 인해 이어지는 다음 심화 2단계 진영인 $M_2$ 모델 전진 요새를 수립(so $M_2$) 할 때도 태생적 한계 원칙상 무조건 아까 이미 뽑아뒀던 고인물 $X_1$ 변수 녀석의 멱살을 잡고 동반 유지 탑승(must also contain $X_1$ together with) 시킨 채, 거기다가 부가적인 잉여 변수(one additional variable) 한 명을 억지로 끼워 올리는 식(one additional variable)으로만 모델이 거대화 성장 확장(stepwise) 될 수밖에 없는 맹점을 내포 안고 지니고(forward stepwise selection must maintain) 있기 때문입니다.

Table 6.1, which shows the first four selected models for best subset and forward stepwise selection on the Credit data set, illustrates this phenomenon.
