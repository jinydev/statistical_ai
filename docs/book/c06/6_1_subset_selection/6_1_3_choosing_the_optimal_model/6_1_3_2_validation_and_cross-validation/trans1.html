Validation and Cross-Validation

검증 및 교차 검증 (Validation and Cross-Validation)

As an alternative to the approaches just discussed, we can directly estimate the test error using the validation set and cross-validation methods discussed in Chapter 5. We can compute the validation set error or the cross-validation error for each model under consideration, and then select the model for which the resulting estimated test error is smallest. This procedure has an advantage relative to AIC, BIC, $C_p$ , and adjusted $R^2$ , in that it provides a direct estimate of the test error, and makes fewer assumptions about the true underlying model. It can also be used in a wider range of model selection tasks, even in cases where it is hard to pinpoint the model degrees of freedom (e.g. the number of predictors in the model) or hard to estimate the error variance $\sigma^2$ . Note that when cross-validation is used, the sequence of models $\mathcal{M}_k$ in Algorithms 6.1–6.3 is determined separately for each training fold, and the validation errors are averaged over all folds for each model size $k$ . This means, for example with best-subset regression, that $\mathcal{M}_k$ , the best subset of size $k$ , can differ across the folds. Once the best size $k$ is chosen, we find the best model of that size on the full data set. 방금 논의한 접근법들의 대안으로, 우리는 제 5장에서 다룬 검증 세트 및 교차 검증 방법을 사용하여 테스트 오차를 직접 추정할 수 있습니다. 고려 중인 각 모델에 대한 검증 세트 오차 또는 교차 검증 오차를 계산하고 그 추정된 테스트 오차가 최소인 모델을 선택합니다. 이 절차는 테스트 오차의 직접적인 추정치를 제공하고 참된 기저 모델에 대해 더 적은 가정을 한다는 점에서 AIC, BIC, $C_p$, 그리고 조정된 $R^2$ 보다 이점을 가집니다. 또한 모델의 자유도(예: 예측 변수 수)를 특정하기 어렵거나 오차 분산 $\sigma^2$ 를 추정하기 힘든 경우 등 더 넓은 범위의 모델 선택 작업에 사용될 수 있습니다. 교차 검증이 이용될 때 알고리즘 6.1–6.3의 모델 시퀀스 $\mathcal{M}_k$ 는 각 훈련 폴드(fold)에 대해 독립적으로 결정되며, 각 모델 크기 $k$ 에 대한 검증 오차들은 모든 폴드에 걸쳐 평균화됨에 주의하십시오. 이는 특정 폴드들 사이에서 최고의 크기 $k$ 부분집합인 $\mathcal{M}_k$ 자체가 서로 다를 수 있음을 의미합니다. 최적 크기 $k$ 가 선택되면, 우리는 전체 데이터 세트에서 해당 크기의 최고 모델을 찾습니다.

In the past, performing cross-validation was computationally prohibitive for many problems with large $p$ and/or large $n$ , and so AIC, BIC, $C_p$ , and adjusted $R^2$ were more attractive approaches for choosing among a set of models. However, nowadays with fast computers, the computations required to perform cross-validation are hardly ever an issue. Thus, cross-validation is a very attractive approach for selecting from among a number of models under consideration. 과거에는 비대한 $p$ 나 $n$ 을 가진 문제에서 교차 검증을 수행하는 것이 계산상 금지될 정도로 무거웠기 때문에 일련의 모델 중 모델을 선택할 때 AIC, BIC, $C_p$ 나 조정된 $R^2$ 가 인기 있는 접근법이었습니다. 그러나 빠르고 강력한 컴퓨터 환경이 갖춰진 오늘날, 교차 검증 수행에 요구되는 연산 과정은 더 이상 거의 제약 사항이 되지 않습니다. 그러므로 교차 검증은 고려 대상 모델들 중 하나를 픽업할 수 있는 아주 효율적이고 매력적인 방식입니다.

In fact, the estimated test error curves displayed in the center and righthand panels of Figure 6.3 are quite flat. While a three-variable model clearly has lower estimated test error than a two-variable model, the estimated test errors of the 3- to 11-variable models are quite similar. Furthermore, if we repeated the validation set approach using a different split of the data into a training set and a validation set, or if we repeated cross-validation using a different set of cross-validation folds, then the precise model with the lowest estimated test error would surely change. In this setting, we can select a model using the one-standard-error rule . We first calculate the standard error of the estimated test $\text{MSE}$ for each model size, and then select the smallest model for which the estimated test error is within one standard error of the lowest point on the curve. The rationale here is that if a set of models appear to be more or less equally good, then we might as well choose the simplest model—that is, the model with the smallest number of predictors.
