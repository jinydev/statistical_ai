# 통계적 학습 (Statistical Learning) 번역 가이드 (통번역 3단계 구조)

이 가이드는 `An Introduction to Statistical Learning`의 마크다운 번역 시 유지해야 하는 3단계(문서 분할) 파일 시스템과 번역 규칙을 정의합니다. `c01`의 번역 스타일을 기준으로 작성되었습니다.

## 0. 시스템 개요: 3단계 파일 구조 (index -> trans1 -> trans2)

학습자의 다양한 필요를 충족하기 위해 단일 문서를 3개의 파일 분할 시스템으로 운영합니다.

1. **`index.md` (영어 원문)**
   - 영어 원문의 단락(Paragraph) 구조와 레이아웃을 책(PDF) 원본과 동일하게 그대로 보존합니다.
   - **이 파일에는 절대 한글 번역문이 포함되어서는 안 됩니다. 순수 영어 원문으로만 표시합니다.** 
   - 최상단에 `trans1.md`(직역본)으로 이동하는 안내 링크구문만 예외적으로 한글을 허용합니다.
   - *예시*: `> 💡 **학습 팁:** 영어 원문 읽기가 어렵다면? 한 줄씩 나란히 번역된 [📖 직역본 보기](./trans1.html)를 추천합니다!`

2. **`trans1.md` (1:1 교차 직역본 - Alternating Bilingual)**
   - 원본 문장을 **마침표(.)를 기준으로 완벽하게 문장 단위로 분리**하여 한 줄씩 표시합니다. 여러 문장을 하나의 문단으로 뭉뚱그려 번역하는 것을 엄격히 금지합니다.
   - **절대 직역**: 영어 표현 그대로 최대한 직역하여 영문에서 표현되는 의미가 원래 의도 그대로 전달되게 해야 합니다. 결코 과장하거나 수식어를 덧붙이는 형태의 의역(paraphrase)을 하지 마십시오.
   - 한 줄의 영어 문장이 오고, 바로 다음 줄에 해당 문장의 한글 직역이 와야 합니다.
   - 최상단에 `trans2.md`(의역/해설본)으로 이동하는 안내 링크를 추가합니다.

3. **`trans2.md` (쉬운 해설 및 스토리텔링 의역본)**
   - `trans2.md` 역시 **`trans1.md`와 구조적으로 동일하게 마침표 기준으로 문장 단위 분리를 유지**해야 합니다. 즉, 영어 원문 한 줄을 그대로 복사해 오고, 그 바로 아래 줄에 한글 번역을 기입합니다.
   - 단, `trans1.md`가 뻣뻣하고 이해하기 힘든 '직역'이라면, `trans2.md`의 한글 부분은 중학생 이상의 독자가 쉽게 원문의 뉘앙스를 파악할 수 있도록 **다양한 예시와 비유, 친근한 대화체를 동원해 길고 풍부하게 풀어쓴 '의역(Free translation)' 및 '해설'**이어야 합니다.
   - 요약하자면: `trans1.md`를 그대로 복사해 온 뒤, **영어 원문은 그대로 두고 한글 직역 부분만 쉬운 해설로 통째로 갈아 끼우는 방식**을 취합니다.

## 1. 문장 단위 분리 및 1:1 교차 직역 포맷 (`trans1.md` 기준)
- `trans1.md` 작성 시 영어 마크다운 텍스트를 마침표(`.`), 물음표(`?`), 느낌표(`!`) 등을 기준으로 하나의 문장 단위로 쪼갭니다.
- 한 줄의 영어 원문 문장 아래에 빈 줄 없이 바로 번역문을 쓰거나, 혹은 빈 줄 단위로 정확한 1:1 교환을 수행합니다. 여러 문장을 합치지 마십시오.
- **구조 예시**:
  ```markdown
  These tools can be classified as *supervised* or *unsupervised*.
  이러한 도구들은 *지도 학습* 혹은 *비지도 학습*으로 분류될 수 있습니다.

  Broadly speaking, supervised statistical learning involves building a statistical model for predicting, or estimating, an output based on one or more inputs.
  크게 말해서, 지도 통계 학습은 하나 이상의 입력을 기반으로 출력을 예측하거나 추정하기 위한 통계 모델을 구축하는 것을 포함합니다.
  ```

## 3. 제목 및 헤더 (Headers)
- 헤더명 역시 1:1 교차 직역하며(`trans1.md`), 헤더 레벨(`#`, `##` 등)은 동일하게 유지해야 합니다.

## 4. 마크다운 문법 보존 (Preserving Markdown Syntax)
- **서식 유지**: 굵게(`**bold**`), 기울임(`*italic*` 등), 인라인 코드(`` `code` ``), LaTeX 수식 표기(`$math$`, `$$math$$`) 등은 원문과 동일하게 한글 번역문에서도 보존해야 합니다.
- **기술 용어 표기**: 중요한 변수명이나 데이터셋 이름, 또는 명확한 전달이 필요한 용어의 경우 인라인 코드 또는 원어를 병기합니다. (예: `` `wage(임금)` ``)

## 5. 이미지 및 캡션 (Images & Captions)
- **이미지 태그**: 이미지를 삽입하는 마크다운(`![alt](./img/...)`)은 영어/한글 교차 없이 한 번만 삽입합니다. (원본 유지)
- **캡션 번역**: 이미지 하단에 굵은 글씨(`**FIGURE 1.1.**`)와 이탤릭체로 구성되는 캡션 텍스트는 1:1 교차 방식과 동일하게 영어 캡션 작성 후 한글 캡션으로 번역하여 배치합니다.

## 6. 프론트매터 및 하위 목차
- 모든 파일 상단의 `---` 로 둘러싸인 YAML 프론트매터(Frontmatter)는 번역하지 않고 원본 그대로 유지합니다.
- `## Sub-Chapters (하위 목차)` 부분은 기존에 작성된 틀을 유지합니다.

## 7. 페이지 이동 네비게이션 (이전/다음 읽기 링크)
- 각 파일(`index.md`, `trans1.md`, `trans2.md`)의 **최상단(Frontmatter 직후)과 최하단**에 다음과 같이 이전 챕터와 다음 챕터로 이동하는 네비게이션 링크를 추가해야 합니다.
- **동일한 번역 레벨 단위의 링크 이동 원칙**:
  - `index.md` (영어) 페이지에서는 이전/다음 챕터의 **`index.html`**로만 이동하도록 링크를 겁니다.
  - `trans1.md` (직역) 페이지에서는 이전/다음 챕터의 **`trans1.html`**로만 이동하도록 링크를 겁니다.
  - `trans2.md` (의역) 페이지에서는 이전/다음 챕터의 **`trans2.html`**로만 이동하도록 링크를 겁니다.
- **구조 예시**:
  ```markdown
  [< 이전 챕터명](../이전폴더경로/index.html) | [다음 챕터명 >](../다음폴더경로/index.html)
  ```
  *(각 번역본 문서의 목적성에 맞추어 `index.html` 부분을 `trans1.html`, `trans2.html`로 치환하여 작성합니다.)*
