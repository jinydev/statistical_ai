# 6.4.3 고차원에서의 회귀 (Regression in High Dimensions)

It turns out that many of the methods seen in this chapter for fitting _less flexible_ least squares models, such as forward stepwise selection, ridge regression, the lasso, and principal components regression, are particularly useful for performing regression in the high-dimensional setting. Essentially, these approaches avoid overfitting by using a less flexible fitting approach than least squares.
이 장에서 살펴본 전진 단계 선택법(forward stepwise selection), 능선 회귀(ridge regression), 라쏘(lasso), 그리고 주성분 회귀(principal components regression)와 같이 _덜 유연한(less flexible)_ 최소 제곱 모델을 적합시키기 위한 여러 방법들이 고차원 설정에서 회귀를 수행할 때 특히 유용하다는 사실이 밝혀졌습니다. 본질적으로 이러한 접근법들은 표준 최소 제곱법보다 덜 유연한 적합 방식을 구사함으로써 과적합을 피합니다.

Figure 6.24 illustrates the performance of the lasso in a simple simulated example. There are $p = 20, 50, \text{or } 2,000$ features, of which 20 are truly associated with the outcome. The lasso was performed on $n = 100$ training observations, and the mean squared error was evaluated on an independent test set. As the number of features increases, the test set error increases. When $p = 20$, the lowest validation set error was achieved when $\lambda$ in (6.7) was small; however, when $p$ was larger then the lowest validation set error was achieved using a larger value of $\lambda$. In each boxplot, rather than reporting the values of $\lambda$ used, the _degrees of freedom_ of the resulting lasso solution is displayed; this is simply the number of non-zero coefficient estimates in the lasso solution, and is a measure of the flexibility of the lasso fit. Figure 6.24 highlights three important points: (1) regularization or shrinkage plays a key role in high-dimensional problems, (2) appropriate tuning parameter selection is crucial for good predictive performance, and (3) the test error tends to increase as the dimensionality of the problem (i.e. the number of features or predictors) increases, unless the additional features are truly associated with the response.
그림 6.24는 단순한 시뮬레이션 예제에서 라쏘(lasso)의 성능을 예시합니다. $p = 20, 50$, 또는 $2,000$개의 특성이 존재하며, 이 중 20개만이 실질적으로 결과 변수와 관련성이 있습니다. 라쏘는 $n = 100$개의 훈련 관측치상에서 수행되었으며, 평균 제곱 오차(mean squared error)는 독립적인 테스트 세트를 통해 평가되었습니다. 포함되는 특성의 수가 늘어날수록 테스트 세트 오차도 함께 증가합니다. $p = 20$일 때는 (6.7) 식의 $\lambda$가 작을 때 검증 세트 오차가 가장 낮게 도출되었습니다. 그러나 $p$가 더 클 경우에는 $\lambda$를 더 큰 값으로 사용할 때 최저 검증 세트 오차를 달성할 수 있었습니다. 각 상자 그림에서는 사용된 $\lambda$ 값을 보고하는 대신 도출된 라쏘 해의 _자유도(degrees of freedom)_ 가 표기되어 있습니다. 이는 라쏘 해에서 계산된 0이 아닌 계수 추정치의 단순 개수를 의미하며 라쏘 적합의 유연성을 가리키는 척도이기도 합니다. 그림 6.24는 다음 세 가지 중요한 사항을 강조합니다: (1) 고차원 문제에서는 규제(regularization) 즉 축소(shrinkage)가 핵심적인 중추 역할을 합니다. (2) 좋은 예측 성능을 거두려면 적절한 튜닝 파라미터 선택이 무척 필수적입니다. 그리고 (3) 만약 새롭게 추가되는 특성 변수들이 반응 변수와 실질적인 관련성이 있는 것이 아니라면 문제의 차원(즉, 특성 혹은 예측 변수의 개수)이 늘어날수록 테스트 평갓값의 오차율도 함께 늘어나는 경향을 보입니다.

The third point above is in fact a key principle in the analysis of high-dimensional data, which is known as the _curse of dimensionality_. One might think that as the number of features used to fit a model increases, the quality of the fitted model will increase as well. However, comparing the left-hand and right-hand panels in Figure 6.24, we see that this is not necessarily the case: in this example, the test set $\text{MSE}$ almost doubles as $p$ increases from 20 to 2,000. In general, _adding additional signal features that are truly associated with the response will improve the fitted model_, in the sense of leading to a reduction in test set error. However, adding noise features that are not truly associated with the response will lead to a deterioration in the fitted model, and consequently an increased test set error. This is because noise features increase the dimensionality of the problem, exacerbating the risk of overfitting (since noise features may be assigned nonzero coefficients due to chance associations with the response on the training set) without any potential upside in terms of improved test set error. Thus, we see that new technologies that allow for the collection of measurements for thousands or millions of features are a double-edged sword: they can lead to improved predictive models if these features are in fact relevant to the problem at hand, but will lead to worse results if the features are not relevant. Even if they are relevant, the variance incurred in fitting their coefficients may outweigh the reduction in bias that they bring.
위의 세 번째 사항은 사실상 이른바 _차원의 저주(curse of dimensionality)_ 로 알려진 고차원 데이터 분석의 핵심적인 기본 원리입니다. 많은 사람은 모델을 적합시키는 데 사용되는 특성의 개수가 늘어나면 늘어날수록, 적합된 모델의 성능 품질 또한 동일하게 증가할 것이라 착각할 수 있습니다. 하지만 그림 6.24의 왼쪽 패널과 오른쪽 패널 수치를 비교해 보면 결과가 인과적으로 일치하지 않음을 알 수 있습니다. 이 사례의 경우만 하더라도, $p$가 20에서 2,000으로 증가함에 따라 테스트 세트 $\text{MSE}$가 거의 두 배 가까이 치솟기 때문입니다. 일반적으로 테스트 세트 오차를 줄인다는 의미에서 _반응 결과 종점과 실질적으로 상관있는 추가 신호 특성 척도들을 추가 편입하는 것은 적합된 모델의 성능을 향상할 것_ 임이 맞습니다. 그러나 반응 요인 지수와 실질적인 아무런 상관이 없는 소음(noise) 특성들을 모델에 무분별하게 추가하는 행위는 결국 적합된 모델 구조의 치명적인 악화를 초래하여 결과적으론 테스트 세트 오답 한계를 크게 높이는 재앙을 낳습니다. 왜냐하면 이런 소음 특성들은 테스트 세트 오차 감소 측면에서의 그 어떤 잠재적 수확도 보장하지 못한 채 (실제 훈련 세트상에서 우연의 일치로 반응 종점과 관련이 있는 것처럼 착각될지 모를 0이 아닐 계수가 할당될 위험이 존재하므로) 과적합 위험성을 걷잡을 수 없이 악화시켜 버리기 때문입니다. 결과적으로 수천수만에 달하는 막대한 특성 지표 측정 수집을 대량 가능케 만들어 주는 이 신기술 구조는 일종의 양날의 검 임이 여기서 증명됩니다. 이 확보 특성들이 당장 닥쳐 직면한 해결 과제와 진정으로 연관되어 있다면 훌륭히 향상된 예측 모델 성과로 맺어질 수 있지만 만약 특성들이 무언가 목적과 무관하게 관련이 없다면 지독하게 나쁜 악의 결과로 되돌아올 것입니다. 심지어 나아가 이 변수들이 관련이 있을 최적의 경우라도 계수를 적합하는 과정에서 우발적으로 초래되는 분산이 자칫 모형이 가져다준 편향의 축소 여부를 능가해 버릴 수도 있습니다.
