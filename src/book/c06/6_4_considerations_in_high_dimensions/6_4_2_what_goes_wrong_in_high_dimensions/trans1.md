# 6.4.2 고차원에서 발생하는 문제점 (What Goes Wrong in High Dimensions?)

In order to illustrate the need for extra care and specialized techniques for regression and classification when $p > n$, we begin by examining what can go wrong if we apply a statistical technique not intended for the high-dimensional setting. For this purpose, we examine least squares regression. But the same concepts apply to logistic regression, linear discriminant analysis, and other classical statistical approaches.
$p > n$일 때 회귀 및 분류 기법 적용에 왜 각별한 주의와 특수한 기법이 필요한지 설명하기 위해, 고차원 세팅 환경을 의도하지 않고 설계된 통계 기법을 적용했을 때 발생할 수 있는 문제점들을 하나씩 살펴보겠습니다. 이를 위해 우리는 우선 최소 제곱 회귀 기법을 살펴봅니다. 하지만 로지스틱 회귀나 선형 판별 분석 같은 기타 고전적 통계 접근법에서도 동일한 개념상 문제들이 똑같이 발생합니다.

When the number of features $p$ is as large as, or larger than, the number of observations $n$, least squares as described in Chapter 3 cannot (or rather, _should not_) be performed. The reason is simple: regardless of whether or not there truly is a relationship between the features and the response, least squares will yield a set of coefficient estimates that result in a perfect fit to the data, such that the residuals are zero.
특성의 개수 $p$가 관측치 수효인 $n$만큼 거대하거나 심지어 더 큰 경우, 3장에서 배웠던 최소 제곱법은 절대 수행될 수 없습니다 (혹은 수행되어서는 _안 됩니다_). 이유는 확실합니다. 특성 요인들과 최종 반응 변수 간에 실제로 엮인 관계성이 존재하는지 여부와 상관없이, 최소 제곱법은 수치상의 잔차(residuals)가 모두 0으로 수렴하도록 데이터를 무조건 완벽하게 적합시키는 계수 추정치 묶음을 산출하기 때문입니다.

An example is shown in Figure 6.22 with $p = 1$ feature (plus an intercept) in two cases: when there are 20 observations, and when there are only two observations. When there are 20 observations, $n > p$ and the least squares regression line does not perfectly fit the data; instead, the regression line seeks to approximate the 20 observations as well as possible. On the other hand, when there are only two observations, then regardless of the values of those observations, the regression line will fit the data exactly. This is problematic because this perfect fit will almost certainly lead to overfitting of the data. In other words, though it is possible to perfectly fit the training data in the high-dimensional setting, the resulting linear model will perform extremely poorly on an independent test set, and therefore does not constitute a useful model. In fact, we can see that this happened in Figure 6.22: the least squares line obtained in the right-hand panel will perform very poorly on a test set comprised of the observations in the left-hand panel. The problem is simple: when $p > n$ or $p \approx n$, a simple least squares regression line is too _flexible_ and hence overfits the data.
그림 6.22는 변수 절편을 포함해 단일 특성 $p = 1$을 가진 경우를 두 가지 환경 관측치 조건(관측치가 20개일 때와 2개일 때)으로 나누어 보여줍니다. 관측치가 20개 확보된 경우는 $n > p$ 기본 조건이 성립하여 최소 제곱 회귀선이 데이터를 오차 없이 완벽하게 적합시키지는 못합니다. 대신, 산출된 회귀선은 20개의 관측 점들을 최대한 합리적으로 잘 근사하려고 노력합니다. 반면에 관측치가 단 2개뿐인 환경에서는, 2개뿐인 관측치의 점수값에 관계없이 두 점을 이은 회귀선이 데이터를 언제나 오차 없이 정확하게 적합시킵니다. 데이터에 대한 이러한 완벽한 적합은 거의 예외 없이 끔찍한 데이터 과적합(overfitting)으로 직결되기 때문에 매우 심각한 문제입니다. 즉 고차원 세팅 환경에서는 훈련 세트 데이터를 오차 없이 완벽하게 계산 적합시키는 것이 기능적으로 가능할지 몰라도, 거기서 뽑아낸 결과 선형 모델은 독립적인 새로운 테스트 세트 평가에서는 극도로 최악의 성과를 보일 것이 뻔하며 결과적으로는 전혀 쓸모없는 모델로 전락합니다. 실제로 우리는 이 재앙이 그림 6.22에서 똑같이 발생했음을 확인할 수 있습니다. 우측 패널에서 구한 최소 제곱 회귀선 도출값은 좌측 패널의 독립된 관측치들로 새롭게 이뤄진 테스트 세트에서 형편없는 성과를 보일 것입니다. 결론은 단순합니다. $p > n$ 이거나 $p \approx n$ 체제일 때 최소 제곱 회귀선 기법은 모델을 과도하게 막무가내로 _유연(flexible)_ 하게 만들어 훈련 데이터를 심각하게 과대 적합시킵니다.

Figure 6.23 further illustrates the risk of carelessly applying least squares when the number of features $p$ is large. Data were simulated with $n = 20$ observations, and regression was performed with between 1 and 20 features, each of which was completely unrelated to the response. As shown in the figure, the model $R^2$ increases to 1 as the number of features included in the model increases, and correspondingly the training set $\text{MSE}$ decreases to 0 as the number of features increases, _even though the features are completely unrelated to the response_. On the other hand, the $\text{MSE}$ on an _independent test set_ becomes extremely large as the number of features included in the model increases, because including the additional predictors leads to a vast increase in the variance of the coefficient estimates. Looking at the test set MSE, it is clear that the best model contains at most a few variables. However, someone who carelessly examines only the $R^2$ or the training set $\text{MSE}$ might erroneously conclude that the model with the greatest number of variables is best. This indicates the importance of applying extra care when analyzing data sets with a large number of variables, and of always evaluating model performance on an independent test set.
그림 6.23 도면은 특성의 개수 $p$가 클 때 부주의하게 최소 제곱 기법을 함부로 적용하는 것의 치명적인 문제점과 위험성을 더욱 잘 묘사합니다. 해당 도표는 총 $n = 20$개의 관측치로 시뮬레이션 합성된 데이터상에 1개부터 20개 사이의 다중 특성을 배정해 회귀를 수행했습니다. 이때 투입된 모든 예측 특성 변수들은 애초부터 반응 타겟 변수와는 아무런 관계조차 없는 완전히 무작위적인 값들이었습니다. 그림에서 알 수 있듯 모델에 수록 투입되는 특성의 개수가 점진적으로 1개씩 증가할수록 모형 평가 판정 $R^2$ 수치는 마치 기적처럼 1로 상승하며, 이 상승 궤도와 일치하게 응당 해당 훈련 모델 세트 모델 측정 $\text{MSE}$ 지수는 거짓말처럼 오답이 전혀 없는 마의 0 기점 수치 구간을 향해 점점 파고들어 끝내 안착 축소 감소합니다. _분명 삽입된 모델의 예측 투입 피처 특성들이 결과 측 반응 종점과 한치도 상관없이 완벽하게 이무 관련 무의미한 독립 요소임에도 불구하고 일어나는(even though the features are completely unrelated to the response)_ 파멸적 수리 눈속임 현상입니다. 반면 역으로 독립적인 새로운 차원의 _독립 테스트 검증 세트(independent test set)_ 진영 내에서의 산출 오차 판정 $\text{MSE}$ 실질 도출 통계 지수들은, 투입되는 평가 특성 변수의 수치 증가 단면 삽입할수록 그에 비례해 기하적으로 극단 파괴적인 우주적 수치 범위로 통제 불능 한없이 거대해집니다. 예측 변수 군단을 무작정 계속 끼워 넣는 행위는 필연적으로 도출 측정 계수 추정치의 통제가 불가한 분산(variance) 지수 증폭 확산을 막대 조장해내어 초래하기 때문입니다. 평가 잔차 테스트 측 오차 계산 도출 MSE를 냉정하게 객관적 살피면, 모형이 가장 빛나는 최고의 최적 성능을 찍는 시점 구조는 결국 변수가 허용 통제 제한된 지극 최소 변수만 담은 경우임이 아주 선명히 자명 단결해집니다. 그러나 이런 데이터 이면 모순을 간과한 채 섣불리 얕은 통계 눈가림에 눈멀어 단순히 $R^2$ 지표나 당장 눈앞 편향 훈련 오류 진영 세트 $\text{MSE}$ 평가만을 단면 살피고 거들떠보는 자는, 가장 무수한 변수를 많이 욱여넣은 모델 구조가 세계 최고라는 잘못된 최악 오 판단 돌이킬 결론에 도출 도달할 것입니다. 이 모순 사실은, 막대한 큰 차원 수치 단위 스케일의 대형 개수 단위 여러 특수 다중 복합 변수 변수들을 보유한 거대 관측 데이터 진영 분석 무대상에서 우리가 분석 평가를 치를 관측 수행할 때 반드시 치밀하게 고차원적인 철벽같은 각별한 정밀 극 주의 통계 검사 방식을 곁들여 강제 구사 결성 투입 적용해야 함과 동시에, 항상 무조건 최종 판단 모델 도출 성능 평가는 철저 완벽하게 격리 확보된 별도의 분리 외부 척도 독립 단면 테스트 세트상에서 치러 단행 판별 확정 지어야 함을 가장 크고 강하게 역설 지시해 보여줍니다.

In Section 6.1.3, we saw a number of approaches for adjusting the training set RSS or $R^2$ in order to account for the number of variables used to fit a least squares model. Unfortunately, the $C_p$, AIC, and BIC approaches are not appropriate in the high-dimensional setting, because estimating $\hat{\sigma}^2$ is problematic. (For instance, the formula for $\hat{\sigma}^2$ from Chapter 3 yields an estimate $\hat{\sigma}^2 = 0$ in this setting.) Similarly, problems arise in the application of adjusted $R^2$ in the high-dimensional setting, since one can easily obtain a model with an adjusted $R^2$ value of 1. Clearly, alternative approaches that are better-suited to the high-dimensional setting are required.
6.1.3절에서 우리는 최소 제곱 모델을 적합시키는 데 사용된 변수의 개수를 고려하여 훈련 세트 $\text{RSS}$ 척도나 $R^2$ 점표를 보정해주는 수많은 여러 접근법을 살펴보았습니다. 안타깝게도 $C_p$, $\text{AIC}$, 그리고 $\text{BIC}$ 기법들은 고차원 환경 설정에서는 결코 적합하지 않습니다. 왜냐하면 고차원에선 분산 추정치 $\hat{\sigma}^2$를 계산하는 것 자체가 매우 문제적이기 때문입니다. (일례로, 3장에서 배웠던 $\hat{\sigma}^2$ 공식을 이 고차원 세팅 환경에 억지로 대입해 적용하면 추정치가 무조건 $\hat{\sigma}^2 = 0$이 도출됩니다.) 마찬가지로, 고차원 세팅에서 단순히 조정된 $R^2$ 척도를 적용하는 것 또한 큰 문제가 발생합니다. 왜냐하면 변수를 욱여넣어 누구든 쉽게 조정된 $R^2$ 값이 1인 거짓 완벽 모형을 얻어낼 수 있기 때문입니다. 결과적으로 고차원 진영 세팅 환경에 훨씬 더 잘 부합하고 어울리는 새로운 대안적 접근 방식들이 분명하게 요구됩니다.
