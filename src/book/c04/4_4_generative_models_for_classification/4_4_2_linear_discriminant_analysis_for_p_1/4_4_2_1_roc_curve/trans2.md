# **ROC Curve**
# **4.4.2.1 ROC 곡선 (수신기 조작 특성 곡선 - 삽질과 대박의 경계선)**

![Figure 4.8](./img/4_8.png)

**FIGURE 4.8.** _A ROC curve for the LDA classifier on the_ `Default` _data. It traces out two types of error as we vary the threshold value for the posterior probability of default. The actual thresholds are not shown. The true positive rate is the sensitivity: the fraction of defaulters that are correctly identified, using a given threshold value. The false positive rate is 1-specificity: the fraction of non-defaulters that we classify incorrectly as defaulters, using that same threshold value. The ideal ROC curve hugs the top left corner, indicating a high true positive rate and a low false positive rate. The dotted line represents the “no information” classifier; this is what we would expect if student status and credit card balance are not associated with probability of default._
**그림 4.8.** `Default` 파산 데이터 구역에 투입된 치안 판사 'LDA 분류기'가 그려낸 그 유명한 ROC 곡선 그래프입니다. 이 곡선은 우리가 탐지 레이더의 과민성 스위치(파산 확률 커트라인 수치)를 빙글빙글 돌리며 조작 세팅값을 바꿀 때마다 널뛰기하는 두 가지 상충하는 치명적 에러들의 피 말리는 변동 궤적을 낱낱이 추적해 도면으로 그려 보여줍니다. 구체적으로 진짜 세팅된 커트라인 수치 자체는 화면에 노출되지 않습니다. 세로축의 '진짜 양성 비율(True positive rate)' 은 바로 전의 챕터에서 그토록 갈구했던 **'민감도(Sensitivity)'** 즉, 현장 잠적한 진짜 파산 빌런 타깃 놈들 중 제대로 멱살 잡아 검거해 낸 짜릿한 적중 구속 족집게 비율입니다. 반면 가로축으로 길게 누운 '거짓 양성 비율(False positive rate)' 은 1에서 **특이도**를 뺀 값으로, 죄도 빚도 없는 멀쩡한 무고한 시민 중 어이없이 오발탄 맞고 파산자로 누명 쓴 안타까운 눈물의 오진 체포 비율을 뜻합니다. 세상에서 제일 착하고 완벽한 궁극의 최강 이상적인 환상의 ROC 치안 곡선은 그냥 저 네모 박스의 왼쪽 가장 위쪽 꼭대기 모서리에 빈틈없이 숨 막히게 착 달라붙어 있는(hugs) 모양새를 자랑해야 합니다. (그건 억울한 시민 체포는 1도 없고 거짓 양성 제로, 범인은 단 한 명도 안 놓치고 100% 싹 다 잡은 완벽한 검거율 진짜 양성 극대의 환상 유토피아 상태를 뜻하니까요!) 가운데 대각선 점선은 뭐냐고요? 그건 정보를 모르는 멍청한 바보 찍기 기계, 즉 고객 스펙이나 통장 빚 잔고 단서가 이 사람 파산 가능성과 아무 연관 짝에도 없는 그냥 쓰레기 단서 데이터일 때 랜덤 찍기로 나올 수수깡 같은 50점짜리 무쓸모 모델 곡선입니다.

---

...marized over all possible thresholds, is given by the _area under the (ROC) curve_ (AUC).
(앞장의 문장 끝에서 이어짐) ...이렇게 임계치 허들을 최저 바닥 0에서 최고 100까지 올렸다 내렸다 별 난리 조작 발광을 피우며 찍어 다 긁어모아 도출한 그 곡선의 전반적인 최종 통합 판결 전투력 종합 측정 등급 점수는 아주 유식하게, 저 그려진 둥근 궤적 **(ROC) 곡선 아래 바닥에 깔린 총 면적의 땅 크기 (Area Under the Curve, 줄여서 AUC)** 점수로 아주 깔끔하게 일괄 계산 정산 부여됩니다.

An ideal ROC curve will hug the top left corner, so the larger area under the ROC curve the better the classifier.
방금 말했듯, 가장 최고로 이상적이고 환상적인 일등 ROC 곡선 기계의 궤적은 그래프 박스의 왼쪽 상단부 모서리 천장 구석을 아주 거머리처럼 찰싹 끌어안고(hug) 찐득하게 바짝 엉겨 붙어 있어야 합니다. 따라서 당연히 그 그려진 곡선 울타리 궤적 아래로 쭈욱 까맣게 색칠되는 넓은 땅바닥 AUC 면적 땅 크기가 광활하고 무지막지하게 넓을수록 그 예측 분류 기계 모델 장치는 칭찬받아 마땅한 더 정교하고 똑똑한 최고의 감별 훌륭한 녀석임을 대변 보증합니다.

For this data the AUC is 0.95, which is close to the maximum of 1.0, so would be considered very good.
우리 예시 파산 타깃 `Default` 데이터 무대에서 뛰고 채점받은 এই LDA 녀석의 AUC 면적 스코어 점수는 무려 광활한 대지 **0.95**를 달성했습니다. 이는 이 우주 AUC 시험 점수판의 절대 만점이자 최대 상한 치인 1.0 극한 점수 천장에 거의 코앞까지 바짝 근접해 도달한 무시무시한 기표 위용입니다! 그러니 우린 이 기계를 가리켜 "야, 이거 물건 엄청 성능 최고다 기가 막힌 모델이다!"라고 아주 당당하게 극찬 인정 간주(considered very good)해 버리면 그만입니다.

We expect a classifier that performs no better than chance to have an AUC of 0.5 (when evaluated on an independent test set not used in model training).
만약 어떤 분류기라는 게 동전 던지기나 눈 감고 부도 예측 다트 던져서 몽둥이질 찍어대는 멍청한 짐승 침팬지 수준의 우연 확률(chance) 통계보다 단 1도 더 나을 것 없는 허접 쓰레기 무지랭이 찍기 기계 장치라면, 우리는 대충 이 녀석의 체면치레 AUC 면적 성적표 전투력이 꼴랑 반 토막 점수인 0.5 (전체 사각형 면적의 절반) 타점일 거라고 기대하고 취급 비판합니다. (물론 이 점수는 훈련 치트키 보지도 컨닝도 못한 아주 생판 독립된 무균 낯선 새로운 현장 테스트 고사장 판 세트에서 시험 평가되었을 때의 진짜배기 팩트 점수 기준입니다.)

ROC curves are useful for comparing different classifiers, since they take into account all possible thresholds.
그래서 이 ROC 곡선 궤적 그래프라는 훌륭한 시각 도판 화면 장치는, 시장에 널려 뛰노는 수많은 다른 출신 종류의 경쟁 분류 기표 예측 모델들을 가져다 앉혀놓고 서로 "누가 누가 더 전투력이 센가?" 한 판 대결 비교 평가 판정 채점하는 데 정말 기가 막히게 유용한 도구 잣대입니다. 왜냐하면 기계마다의 특정 한 가지 단편 커트라인 기준 잣대로만 편파 평가하지 않고, 발작 반응 튜닝 스위치를 바닥 0부터 100 천장 끝까지 조율 돌려가면서 터지는(all possible thresholds) 기계 반응 모든 스펙 스펙트럼 궤적 체력을 모조리 감안(take into account)해서 포괄적 전체 곡선 넓이 전투력 체력을 공정하게 다 끌어내 까발려 합산해 종합 평가해 주거든요.

It turns out that the ROC curve for the logistic regression model of Section 4.3.4 fit to these data is virtually indistinguishable from this one for the LDA model, so we do not display it here.
충격적인 스포일러 재미있는 진실 에피소드 하나 까발리자면, 우리가 한참 이전 챕터 (섹션 4.3.4) 에서 피 터지게 공부했던 저 유명한 '로지스틱 회귀 (Logistic Regression)' 녀석 머신 장치를 똑같이 이 파산 데이터 현장판에 끌고 와서 장착 가동 튜닝해 돌려 찍어 올려 그려본 그 녀석의 ROC 곡선 궤적 그림마저도 알아봤더니, 세상에나 지금 우리가 기를 쓰고 만든 이 짝퉁 인간 무기 LDA 녀석의 곡선 라인과 외모 스캔 윤곽 형태가 서로 그냥 육안으로는 도무지 분간 차이가 1도 안 될 정도로 시각 곡선이 기괴하게 딱 겹쳐 완벽히 들러붙어 똑같이 똑같았습니다(virtually indistinguishable). 그래서 교과서 저자들도 어이없어서 "야, 이거 그림 똑같은데 굳이 복붙 낭비해서 여기 나란히 디스플레이 펼쳐 그릴 필요 있냐?" 라며 쿨하게 로지스틱 기계 곡선 그림은 그냥 아예 버리고 패스 삭제 생략 안 그렸습니다.

As we have seen above, varying the classifier threshold changes its true positive and false positive rate.
지금껏 우리가 침 튀겨가며 목도 파악 구경했듯, 기동 분류 탐지 기계 장치의 과민 반응 저항 스위치라 불리는 임계값 컷트라인 허들 제한선(threshold)을 쪼잔하게 이리저리 오르락내리락 튜닝 변조(varying) 조작 뒤트는 작업질 수술은, 필연적으로 엄청난 스노우볼 대가를 불러일으켜 우리가 그토록 환호 찬양 갈망하는 짜릿한 정답 범인 색출 적중 검거 타점률인 '진짜 양성 비율(true positive rate, 적중 검거)'과 반대로 눈물 쏙 빼는 억울한 선량 시민 무고 헛다리 체포 오판 대재앙 타점 기표인 '거짓 양성 비율(false positive rate, 오판 포획)' 두 지표의 무게 중심 수치 점수를 미친 듯 요동치고 격노 변화 변경(changes) 시킵니다.

These are also called the _sensitivity_ and one minus the _specificity_ of our classifier.
이 길고 긴 이름의 두 요동 점수 지표들, 헷갈리니까 좀 학술 있어 보이는 전문 간판 칭호 은어로 바꿔 퉁 호출 부르면, 진짜 타깃 빌런 검거 타점률은 기계 레이더의 훌륭 보증 간판인 우리 측 분류기의 예민 **민감도(sensitivity)** 라고 부르고, 눈물 오판 생사람 잡기 검거 타점률은 철통 무고 방벽 수비 구멍 파쇄 점수인 '1 빼기 **특이도(specificity)** ' 라는 이름 암호로 대 변환 일컬어져 매칭 통용 불리고 칭호를 장식 받습니다.

Since there is an almost bewildering array of terms used in this context, we now give a summary.
솔직히 말해 진짜 고백하자면, 이 통계학 불량 검거 분류 판독의 전공 세계 맥락 판에 진입해 들어오면 정말 혼이 쏙 빠질 정도로 머리가 어지럽고 미칠 듯이 난잡 복잡 쏟아지는(bewildering) 동의어 용어 칭호 단어 덩어리 대 환장 파티 폭격 배열(array) 라인업에 정신을 잃을 지경일 것입니다. 그래서 통계 학자들도 양심은 있는지 불쌍한 우리 구경꾼 뉴비들을 위해 여태 혼동 판을 아래처럼 단칼에 표로 딱 축약 정리 압축해 요원 요약(summary) 단판 선물 제공 투입을 해 줍니다.

Table 4.6 shows the possible results when applying a classifier (or diagnostic test) to a population.
아래 표 4.6 마법 도표는 우리가 개발한 분류기 인공지능 장치(또는 코로나 불량 바이러스 찌르는 의학 진단 키트 센서 막대기 테스트든 뭐든 다 똑같은 논리입니다!)를 불특정 다수의 대중 인간 표본 덩어리 인구 모집단에 냅다 무자비하게 팍 투입 돌려 돌렸을 때 짜게 쪼개져 튕겨 파생 발생 될 수 있는 모든 4가지 양단의 진위 결판 매칭 구동 경우의 수 운명 결과 조합판을 모두 그려 까발려 보여줍니다.

To make the connection with the epidemiology literature, we think of "+" as the "disease" that we are trying to detect, and "-" as the "non-disease" state.
의학 병리학 (역학, epidemiology) 세계 최전선 전공 서적 의사들 문헌 세계관과의 개념 논리 탑재 호환 연결 퓨전 매칭 세팅을 자연스럽게 돕기 위해 룰을 약속하죠. 이제부터 우리는 저 수식 속 딱딱한 십자 문자 표시 **"+"** 표지판을 우리가 그렇게 피 터지게 치안 레이더로 기를 쓰며 검거 감지(detect) 탐색하고자 하는 몹쓸 바이러스 확진 오명 빌런 불량 표적 상태 즉 **"질병 감염 파산 전과자 놈(disease)"** 으로 철저히 등치 치환 생각하겠습니다. 그리고 깨끗한 마이너스 짝대기 **"-"** 기호는 착하고 성실 건강 무고한 **"질병 안 걸린 멀쩡 청정 선량한 놈(non-disease)"** 의 착한 결백 방면 상태 기호라고 세팅 깔고 편히 생각합시다.

To make the connection to the classical hypothesis testing literature, we think of "-" as the null hypothesis and "+" as the alternative (non-null) hypothesis.
혹시 당신이 골수 통계학도라 고리타분한 옛날 부머 고전 통계학 가설 검정 논문을 파고드는 학구파 문헌 연결을 갈망한다면, 저 "-" 빼기 기호는 그냥 통계학의 너무나 흔해 빠진 노잼 기본 상식 가설인 그 유명한 귀무 가설(null hypothesis, 무 기각 기본 상태)로, 반대 "+" 부호는 무언가 유의미 엄청 특이 결단 큰 건 발견 특종 사태 기 현상 대 징후 발생의 대립 가설(alternative, non-null) 로 등급 간주해 변환 착각 치환 연결 부합 세팅 매칭 돌려 생각하셔도 딱 논리 찰떡 호환됩니다.

In the context of the `Default` data, "+" indicates an individual who defaults, and "-" indicates one who does not.
다시 우리가 푸는 카드 빚 파산 폭탄 `Default` 실습 데이터 수조 환경 문맥 동네 구덩이판으로 눈높이를 싹 낮춰 비유 팩트 복귀 환산 적용하자면, 여기서 붉은 적색 꼬리표 훈장 **"+" 표식은 당연히 야반도주 파산(default)을 대박 저지른 진성 빌런 도주 인간 개체**를 정조준 타깃 찌르고 명백 지목 지칭하며, **"-" 표식은 꼬박꼬박 빚 안 밀리고 잘 갚고 이탈 파산 안 하는 정상 꿀 착한 고객 인간** 결백 무고 상태를 가리키는 셈입니다. 

| | | True class (신의 눈으로 본 절대 진위 클래스 진실) | | |
