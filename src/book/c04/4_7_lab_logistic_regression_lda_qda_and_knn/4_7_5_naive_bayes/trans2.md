# 4.7.5 Naive Bayes
# 4.7.5 나이브 베이즈 무기 가동: "대충 끼워 맞춰도 명타는 냅니다!"

Next, we fit a naive Bayes model to the `Smarket` data.
자, 이번엔 이름부터가 '순진해 빠진' 무기, 바로 앞뒤 안 가리고 무대포로 끼워 맞추는 기적의 확률 찍기 도구인 **나이브 베이즈(naive Bayes)** 모의 예측 모델 머신을 이 골치 아픈 `Smarket` 주식 관측 데이터 표본 집단에 강제 투입 투척하여 모의 적합시켜 쾅쾅 박아보겠습니다. 성능이 과연 얼마나 나올까요?

The syntax is similar to that of `LDA()` and `QDA()`.
게다가 핵이득! 이 무기를 다루는 파이썬 언어 기동 제어 스위치 조작 구문(syntax) 코딩 사용 방식 템플릿 역시, 방금 전 우리가 신나게 갈겨댔던 구형 무기 `LDA()` 모델 조작 작동 버튼과, 최신 곡선 기동 장비인 `QDA()` 구동 체제의 제어 패널 규격과 거의 매우 동일하게 똑같이 쌍둥이처럼 일치 호환 반복 유사합니다. 

By default, this implementation of the naive Bayes classifier models each quantitative feature using a Gaussian distribution.
이놈의 설계 특이점 주의! 파이썬 공구 패키지 내부 코드 작동 기본 디폴트 설정 체제(By default)상으로, 해당 이 나이브 베이즈 통계 분류기라는 놈이 훈련 전산 데이터를 씹어먹을 때의 독특한 논리 시스템 구현체(implementation) 방식이 있습니다. 이놈은 무식하게 한 방에 데이터를 통으로 엮지 않고, 분석 투입된 징후 조각들(`Lag1`, `Lag2` 등등)인 예측 특징(quantitative feature) 개별 데이터 파편 조각 하나하나 개별 단위 무더기들을 각각 다 따로따로 독립 분리해서 취급하며, 하나하나의 단서 묶음들을 죄다 가장 우아하고 예쁜 좌우 대칭 종 모양의 수학적 통계 산 형태인 **가우시안 확률 정규 분포(Gaussian distribution)** 통계 이상향 곡선 모형에 무조건 모조리 찍어 누르며 강압 억지 모델링하여 자체 룰의 틀에 욱여넣는 단순하고 오만한 버릇 로직을 디폴트로 갖습니다. (그래야 계산이 초스피드로 빠르거든요!)

However, a kernel density method can also be used to estimate the distributions.
그러나, "야, 주식 데이터가 무슨 완벽히 예쁜 정규 분포 종 모양이야? 울퉁불퉁 엉망진창 현실 세계를 직시해!"라고 일침을 가하고 더 정밀 분석 조작을 요구하고 싶다면? 프로그래머의 조작 자유 의지로 다른 옵션 코드를 세팅 기동 지시해서 통계 변수 내부 곡선 덩치 분포 모형을 아주 뾰족 정밀 계산 추산 획득 및 굴곡 확률 추정(estimate) 파악 작동 변환 하기 위한 구동 옵션으로써, 수학적 현실 점선 굴곡 산점 도출 방식 인 매우 정밀한 고급 **커널 밀도 추정 방법(kernel density method)** 로직 구동 날붙이 도구 또한 얼마든지 스위치를 켜 조작 교체 사용 탑재 변경 운영시킬 수 있는 확장 커스텀 자유 기능을 은밀 지원 무장하고 있습니다.

```python
In [38]: NB = GaussianNB() # 얘도 스탯 수집 장치야! 가우시안 종 모양 모자로 세팅해서 발진 소환!!
NB.fit(X_train, L_train) # 2004년 과거 기출문제로 "훈련(fit) 개시!! 뺑뺑이 돌려라!"
```

```python
Out[38]: GaussianNB() # "훈련 시스템 부팅 딥러닝 뚝딱 완료요!"
```

The classes are stored as `classes_`.
훈련 구동 결과, 목표 타깃 그룹 종속 확률 판정 클래스들('떡상할 놈들', '떡락할 놈들') 두 범주 정보 표상 정보 출석부 기록 명부는 역시 기계 내부에 `.classes_` 란 이름의 내부 은닉 속성 객체 기능 인덱스 단편 공간으로 알뜰하게 파편 백업 분류 저장(stored) 안착 완료됩니다.

```python
In [39]: NB.classes_ # 야 출석부 이름 명부 까봐!
```

```python
Out[39]: array(['Down', 'Up'], dtype='<U4') # 1팀 나락(Down), 2팀 비상(Up)!
```

The class prior probabilities are stored in the `class_prior_` attribute.
해당 그룹 단원 판별 분류 출현의, 힌트 따위 다 빼고 무지성 기본 바닥 등장 비율 통계인 소위 **기본 등장 밑밥 사전 확률들(prior probabilities)** 점유 몫 비 데이터 결괏값들은 모형 내부에 보존 은닉된 개별 특수 조작 확인 속성 저장고 서랍인 `.class_prior_` 독립 배열 속성 구조 탭 공간 안에 은닉 분리 저장 별도 백업 분리 보관됩니다.

```python
In [40]: NB.class_prior_ # 힌트 없이 눈 감고 찍었을 때 기본 배당 스탯 어떻게 돼? 확률 까봐!
```

```python
Out[40]: array([0.49198397, 0.50801603]) # Down 부대 49%, Up 부대 50.8% 지배 중!
```

The parameters of the features can be found in the `theta_` and `var_` attributes.
자, 이제 이 순진한 예측 무기의 가장 중요한 핵심 무기 핵심 통계 힌트 지표! 개입 투입된 조작 단서 투입 힌트 특징(features) 징후 변수(`Lag1`, `Lag2`) 파라미터 핵심 스탯 값들은 각각 따로 파편 분리되어, `.theta_` (각 관측 단서 특성 그룹의 산술 본진 '평균'을 품은 지갑 메모리 속성) 과 바로 이어 보관된 `.var_` (각 관측 단서 특성의 분열 치기, 자체 '분산'을 담은 지갑 메모리 속성) 이라는 기판 통계 2가지 속성값 핵심 서랍 단편 메모리 열람 공간 안에서 분석가가 모두 단적으로 분명히 데이터 기록 확증 열람 파악 단별 개별 발견 탐색 꺼내 도출 조회 열람될 수 있습니다.

The number of rows is equal to the number of classes, while the number of columns is equal to the number of features.
출력 덩어리 구조 보는 법 가이드! 이 출력 결과 표 공간을 들여다볼 때, 차곡차곡 쌓인 2차원 배열 세로 폭 깊이인 **행(rows)** 방향의 층 탑재 개수는 단말 예측 타깃 목표 지목 도출 팀의 타깃 진영 **클래스 갯수 전체 수**(`Down`, `Up` = 2층!) 와 일치 쌍 상응 수치 크기가 동일(equal) 구조를 지닌 한 반면, 이와 반대로 가로 단절 배열 종속 폭 전개 구조인 **열(columns)** 의 옆 파편 길이 배당 공간 숫자는 관측 단서 속성 변동 예측 힌트로 총알 개입 투입 타깃 짚어 넣었던 힌트 타깃 **특징(features) 단일 변수 개수**(`Lag1`, `Lag2` = 2칸!) 의 숫자 폭 길이 스펙 열 기둥 정보와 완벽히 열쇠와 자물쇠처럼 일치 쌍 배열 동일 대응 구조 짝을 맞춥니다.

We see below that the mean for feature `Lag1` in the `Down` class is $0.043$.
아래 계산 관측 표 산출 호출 로그 도출 결괏값 데이터 콘솔 화면에서 우리는 그 끔찍한 나락 하락 도출 상징 그룹 `Down` 종속 분류 클래스 팀 서열 진영 안에서, 바로 첫 번째 투입단 관측 총알 투입 조작 계산 힌트 징후를 담은 예측 단서 부품인 예측 변형 변수 단편 계산 부품 정보인 **과거 단서 `Lag1`** 변수 스탯 통계 정보의 수학적 독립 속성 단위 스탯 개별 분리 평균 스코어 기록(mean) 수치 도출 좌표 지표가, 대략적 산출 계산 결과 산술상 도출 부품 좌표 데이터 평균 스탯인 $0.043$ 수치 파편 값 좌표로 기점 계산 완료 형성 스폰 되었음을 직접 콘솔 관측 확인해 살펴볼 수 있습니다. 

```python
In [41]: NB.theta_ # 각 클래스별 힌트 변수(머신러닝 총알)들의 좌표 중앙 평균값을 다 까봐!!
```

```python
Out[41]: array([[ 0.04279022,  0.03389409], # (Down 팀의) [Lag1 평균, Lag2 평균]! 여긴 플러스네?
                [-0.03954635, -0.03132544]])# (Up 팀의)  [Lag1 평균, Lag2 평균]! 여긴 마이너스 좌표네?
```

Its variance is $1.503$.
그리고 그 옆 전용 서랍인 분산(Variance) 속성을 조회하면? 바로 그 타깃 대상의 덩치를 보여주는 해당 개별 지표 정보인 독립 그룹 단일 집단 종속 공분산 고유 덩치 부풀기 개체 정보 파편인 폭 확률 분산 결괏값(variance) 수치 통계 퍼진 정도 스탯 기록 편차 기록 넓이 치수가 무려 대략적으로 $1.503$ 스탯 위용으로 측정 연산 계산 세팅 됨을 알게 됩니다. 

```python
In [42]: NB.var_ # 이번엔 이 힌트 부품들의 개별 독방 덩치 스펙! 분산 수송 스탯 크기를 까봐!!
```

```python
Out[42]: array([[1.50355424, 1.53246652],   # (Down 팀의 분산 덩치) [Lag1 분산, Lag2 분산]
                [1.51397116, 1.4870335 ]])  # (Up 팀의 분산 덩치)  [Lag1 분산, Lag2 분산]
```

How do we know the names of these attributes?
근데 잠깐, 여기서 초보 분석가 여러분의 아주 순수하고 합리적인 코딩 의문! 도대체 파이썬 쌩초보인 우리가 대체 프로그램 머신 내부 깊숙이 안감에 꽁꽁 컴파일 숨김 되어 다이 캐스팅 은닉된 이러한 파이썬 복잡 기계 장비 내부 비밀 내장 파편 특수 부품 고유 부착 은닉 속성 데이터 객체 변수명 들(`theta_`, `var_` 같은 이상한 용어들!)의 그 은닉된 숨겨진 비밀 암호 이름들(names) 메뉴판 목록 열람표 전체 코드를 스크린 창 외부 껍데기 단에서 그토록 무슨 수로 척척 전격 편하게 어떻게 꿰차고 모조리 다 스탯 호출 단어로 알고 꺼내 쓸 방도 길이 있을까요?! 천재가 아니면 못 하나요?

We use `NB?` (or `?NB`).
절대 아닙니다 꿀팁 공개! 코딩 대답은 심플 명쾌합니다. 우리는 거시적 천재 코딩 스펙이 필요 없이 그냥 아주 단순 원시적 강압 호출 기능인, 파이썬 패키지 내부 강제 속성 취조 특수 헬프 검색 기능 쿼리 취조 호출 명령 조작줄 변형 호출 기능 강제 소환 도움말 호출 단축 폭로 명령 규격 기능인 **`NB?` 파이썬 도움말 질문 쿼리 호출 기능 명령어 투척 (혹은 뒤집은 명령 타격 방식형 변동 문자열 명령 줄 `?NB`)** 를 해당 기계 변수 에 조작 강제 투입 투척 대단히 조준 도출 호출 타격 강압 작동 실행 사용해 버리면 모든 부품 설명서 창이 해킹 열람되어 아주 은밀하고 편리하게 통째로 다 까발려 화면에 쏟아져 나타 납니다!

We can easily verify the mean computation:
또한, 덧붙여 우리는 이렇게 저 시스템 기계 로켓 엔진 모델이 닫힌 폐쇄된 암막 상자 회로 내부 안방 은닉 구석 시스템 동작 속 내부 도출 작동 은직 산술로만 통계 독립적 자체 처리 단일 기작 이행 수행 판단한 저러한 자체 블랙박스 내부 수동 단독 평균 추산 도출법 계산법 연산 측정(computation) 작동 기작 구조 동작 처리 출력 연산 이력을 통계 패키지 기능 없이도 굳이 내가 아주 쉽게 파이썬의 단순 데이터 쪼개기 기술만으로 독자적으로 외부적으로 직접 수동 스크래치 임의 단독 교차 증명 계산하여 단독 기계 오류 의심 수동 검증(verify) 입증 확인 맹점 일치 연산 교차 대조 논리 증명 점검해 낼 수도 있는 강력한 통계 융합 자가 검증 수단 확인 능력이 코딩 데이터에 전면 아주 확실하게 있습니다:

```python
In [43]: X_train[L_train == 'Down'].mean() # "야, 기계 못 믿겠다! 내가 직접 코딩 짠다! X_train 데이터에서 'Down' 정답치 뜬 놈들만 손으로 멱살 잡아 골라내서 직접 mean(평균) 계산 때려봐!!"
```

```python
Out[43]: Lag1    0.042790
         Lag2    0.033894 # 오, 진짜 아까 NB.theta_ 기계가 뽑아낸 맨 윗줄 결과 배열 값이랑 토씨 하나 안 틀리고 소름 돋게 똑같네? 기계 조작 아니었음 검증!
         dtype: float64
```

Similarly for the variance:
유사하게 매우 완벽 상응 동일 원리 조작 증명 작동(Similarly) 교차 증명 핀 포인트 검사 패턴으로써 저 위 도출 통계 집단 증명인 분산(variance) 통계 덩치 데이터 변수 스탯 확증 수치 측정 측면 작동 조작 증명 방식 에제 에 대해서도, 우리는 또다시 한번 개별 독립 분열 입증 수치 산술 일괄 도출 연산 수동 단독 조작을 직접 코딩 타이핑하여 동일 단절 증명 증명 검증 대조 구동 도출 합니다:

