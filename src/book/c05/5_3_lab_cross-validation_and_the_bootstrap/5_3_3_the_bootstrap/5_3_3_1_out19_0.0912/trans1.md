### Estimating the Accuracy of a Linear Regression Model

The bootstrap approach can be used to assess the variability of the coefficient estimates and predictions from a statistical learning method. Here we use the bootstrap approach in order to assess the variability of the estimates for $\beta_0$ and $\beta_1$, the intercept and slope terms for the linear regression model that uses `horsepower` to predict `mpg` in the `Auto` data set. We will compare the estimates obtained using the bootstrap to those obtained using the formulas for $\text{SE}(\hat{\beta}_0)$ and $\text{SE}(\hat{\beta}_1)$ described in Section 3.1.2. 
부트스트랩 접근 타진 방식 기조는 임의의 기계 통계 학습 메커니즘 구동 모델상에서 돌출돼 떨어져 나오는 각종 계수 추정 지표들(coefficient estimates) 및 예측 파생값 결괏치 본연이 지닌 가변성(variability) 널뛰기 이격을 진단 평가(assess)하려는 용도 기반으로 폭넓게 착취 운용(used to)될 수 있습니다. 작금의 대목(Here)에서 우리들은 바로 이 본연의 부트스트랩 접근 시도를 채택 취하여 활용할 셈인데, 이의 목적은 기 출시된 `Auto` 차량 제원 데이터 더미 묶음 정보 위에서 기어코 `horsepower` 인자를 축으로 `mpg` 결괏수치를 측량 견인 예측(predict)코자 도출 산출해 올렸던 일직선 1차 회귀 모듈 함수가 뿜어내고 산출한 구성요소 성분들 즉 $Y$ 절편 위치 축항 및 곡선 편향 기울기 단위체로 지목 추종되는 쌍방의 모수 표적 치수 $\beta_0$ 및 $\beta_1$ 두 녀석이 태생적으로 발산 웅거하고 지닌 편차적 변동성 이탈 변위를 직접 검정 평가(assess) 수리하려는 소임 때문입니다. 향후 우리는 방금 부트스트랩을 매개로 삼아 도출 획득 구득 달성된 개별 모수 쌍의 편차 추산치 수위 결과물을 예전 3.1.2장 도입부에 언급 기입 서술(described) 남겼었던 $\text{SE}(\hat{\beta}_0)$ 와 $\text{SE}(\hat{\beta}_1)$ 측량 공식 체계를 활용해 도출해 건졌던 당대 기록 산출물들(those)과 나란히 상호 대조 왈가왈부 비교(compare) 교차 점검할 것입니다.

To use our `boot_SE()` function, we must write a function (its first argument) that takes a data frame `D` and indices `idx` as its only arguments. But here we want to bootstrap a specific regression model, specified by a model formula and data. We show how to do this in a few simple steps. 
지금 우리가 짠 맞춤 `boot_SE()` 코딩 연산 체제를 매끄럽게 실제 원활 운용 탑승 구동하려면, 사전에 반드시 `D`라고 불리는 데이터 구조 판과 `idx` 조준 주소 일련 지표 인덱스를 쌍끌이 단 두 개의 한정된(only) 순수 인자 요건체(takes)로만 취급해 영접 흡수하는 사전 기초 밑작업 격의 개체 종속 별립 함수(그 자체로 함수 1선발 인자가 될 것임) 하나를 사전에 단독 축조 기입 코딩(write) 완료해 매어두어야 하는 의무 제약이 따릅니다(must). 그런데 작금 닥친 이곳 국면에서 우리들의 희망(want to)은, 오히려 수식(1) 조건과 (2)데이터 구조를 병합 적시(specified) 구명하여 못을 박아 규정 픽스해놓은 철통같은 하나의 특정 회기 예측 모델 통째 모듈체를 거대하게 부트스트랩 빙빙 돌리기 전개 회전시키고픈 포부에 기원합니다. 이어지는 몇 회의 아주 간소 간단한 편의적 코딩 걸음(steps) 처리 편제 절차들을 통해 이런 요구의 실질 구현 달성 메커니즘을 상세히 시연 내보여(show) 증명합니다.

We start by writing a generic function `boot_OLS()` for bootstrapping a regression model that takes a formula to define the corresponding regression. We use the `clone()` function to make a copy of the formula that can be refit to the new dataframe. This means that any derived features such as those defined by `poly()` (which we will see shortly), will be re-fit on the resampled data frame. 
그 시발점(start by)으로 우리는 해당 대응 회기 체계를 뚜렷이 설계 입증 확정(define)코자 그 뼈대 공식 자체를 입력 흡인(takes) 접수하는, 회귀 모형용 뺑뺑이 재추출 기믹 가동 공구로서의 일반 범용 형태성 성향(generic)의 다용도 함수인 `boot_OLS()` 를 백지부터 하나 기입 마련 축조하는 것에서부터 신호탄을 쏩니다. 우리들은 그 안에 내장된 파이썬 기능 공구 `clone()` 명령어 체제 기능을 활용 착취하여 본 공식 규격 본체와 구별되는 거푸집 복사본 사본 껍데기(copy)를 1차 마련해 복제시켜 냄으로써, 그것이 향후 계속 재등장할 낯설고 신규한 새 데이터 판 위에 족족 자유자재 재 장착 접합 피팅(refit) 구사되도록 활로를 열어젖힙니다. 이 같은 선결 처리 체계가 의미(means that) 부여하는 바인죽, 앞으로 우리가 맞닥뜨릴 (가령 잠시 뒤에 조만간 관찰 전개시켜 목격하게 될) 이른바 `poly()` 다항수식 전개항과 같은 작위에 의해 전개 창조되는 일체의 파생 발생 기인 스펙 특징 징후 변동물(derived features)들 역시도 결국에는 전부, 반복 재모집 창출 배포된 당해 신생 표본 풀 프레임 기단 위층에 몽땅 죄다 한 점 예외 없이 다시 재구성 탑재 장착 갱신되어 리-피팅(re-fit) 처리 동화 접수됨이 가능해진다는 사실에 다름 아닙니다.

This is not quite what is needed as the first argument to `boot_SE()`. The first two arguments which specify the model will not change in the bootstrap process, and we would like to _freeze_ them. The function `partial()` from the `functools` module does precisely this: it takes a function as an argument, and freezes some of its arguments, starting from the left. We use it to freeze the first two model-formula arguments of `boot_OLS()`. 
다만 솔직히 까놓고 고백하자면 현시점 저 위에 갓 빚어진 저 조악한 모양새 코딩만으론 여전히 메인 엔진인 `boot_SE()` 기능체 입구의 선두 제1 타겟 인수 자리에 냅다 투입 활용되기에 뭔가 약간 상당히 부적합하고 딱 맞아떨어지지 않는 체적 어불성설 불합리성(not quite what is needed)의 요소가 은은히 깃들어 있습니다. 모델 몸체 자체를 지목 확정 규정 픽싱시켜내는 저 첫 번째 도립된 2쌍의 인자 단위들은 장차 숱하게 무한 궤도로 빙빙 반복될 거대한 부트스트랩 반복 가속 프로세싱 궤도 여정 절차 내내 단 한 치도 융통 전환 변용될(will not change) 요량과 건더기가 전무하며, 고로 차라리 우리들 입장에선 그들의 고정값을 일찌감치 한데 영구 점착(freeze) '얼음' 고착 조치 동결시켜 버리는 편이 낫습니다. 바로 요 부분, 파이썬 부품 `functools` 모듈 상에 안착 보관되어 있는 `partial()` 기믹 함수란 녀석이 그야말로 빈틈 한 치 없이 이 요구 소임 자체를 기깔나고 기가 막히게 세밀 정교히(precisely) 처리 성사해 냅니다: 이 공구 녀석은 거대한 1개의 타겟 함수 자체를 하나의 뭉퉁그려진 대상 인자 속성인 양 삼켜버린(takes) 채로, 그 함수 좌측면(from the left) 변두리에 대기 나열 붙어있는 인자열 단위 구멍 중 절반 대다수 일부 요원들만 쏙 지목해서 얼음 땡 영구 동결 고착 점착 마비(freezes) 처분을 강행 기입해 줍니다. 우리는 바로 이러한 `partial()` 녀석의 절묘한 냉동 스킬 기능을 부려먹음(use)으로써 조금 전 코딩 설계했던 갓난 함수 `boot_OLS()` 몸체의 서두를 여는 선두 2개쌍, 곧 즉 모형과 수식 골자 뼈대를 각각 지목 지시 담당하고 있던 바로 그 두 관문 인자(model-formula arguments)를 사전에 철두철미 차갑게 못 박아 영원 불변 고정 동결(freeze) 조치해버립니다.

The standard error estimates for $\hat{\beta}_0$ and $\hat{\beta}_1$ obtained using the formulas from Section 3.1.2 are 0.717 for the intercept and 0.006 for the slope. Interestingly, these are somewhat different from the estimates obtained using the bootstrap. Does this indicate a problem with the bootstrap? In fact, it suggests the opposite. Recall that the standard formulas given in Equation 3.8 on page 75 rely on certain assumptions. For example, they depend on the unknown parameter $\sigma^2$, the noise variance. We then estimate $\sigma^2$ using the RSS. Now although the formula for the standard errors do not rely on the linear model being correct, the estimate for $\sigma^2$ does. We see in Figure 3.8 on page 99 that there is a non-linear relationship in the data, and so the residuals from a linear fit will be inflated, and so will $\sigma^2$. Secondly, the standard formulas assume (somewhat unrealistically) that the $x_i$ are fixed, and all the variability comes from the variation in the errors $\epsilon_i$. The bootstrap approach does not rely on any of these assumptions, and so it is likely giving a more accurate estimate of the standard errors of $\hat{\beta}_0$ and $\hat{\beta}_1$ than the results from `sm.OLS`.
