# 5.1.3 k-Fold Cross-Validation
# 5.1.3 궁극의 밸런스 패치: K-폴드 교차 검증 (k-Fold CV)

An alternative to LOOCV is _k-fold CV_ . This approach involves randomly dividing the set of observations into $k$ groups, or _folds_, of approximately equal size. The first fold is treated as a validation set, and the method is fit on the remaining $k - 1$ folds. The mean squared error, $\text{MSE}_1$, is then computed on the observations in the held-out fold. This procedure is repeated $k$ times; each time, a different group of observations is treated as a validation set. This process results in $k$ estimates of the test error, $\text{MSE}_1, \text{MSE}_2, \dots, \text{MSE}_k$. The $k$-fold CV estimate is computed by averaging these values:
아까 본 LOOCV의 자비 없는 N번 뺑뺑이 서버 극딜을 제지하기 위해 등장한 가성비 최강의 콤보 스킬, 바로 **_k-폴드 CV (K분할 교차 검증)_** 입니다. 이 녀석의 작전 지시서는 심플합니다. 아까처럼 1명 단위로 고문하는 게 아니라, 전체 학생 데이터 세트를 딱 눈대중으로 공평하게 $k$ 개비의 묶음 파벌(이걸 영어로 _폴드 folds_ 라고 부릅니다) 로 무작위 찢어발깁니다. 1번 폴드를 유배지에 가둬서 "너희들이 첫 번째 검증 모의고사를 쳐라!" 라고 지시한 뒤, 나머지 $k - 1$ 개의 남은 거대 동맹군 폴드 전체를 '훈련소' 로 밀어 넣고 모델을 펌핑(fit) 시킵니다. 그런 다음, 이 훈련된 머신을 첫 빠따 유배지(1번 폴드) 로 끌고 가서 그들이 치르는 모의고사 성적표 $\text{MSE}_1$ 을 땁니다. 자, 다음 턴! 이제 2번 폴드를 모의고사 유배지에 던지고, 방금 전 1번 폴드 포함 나머지 전원은 다시 훈련소로 강제 입소. 이렇게 모든 파벌 조직 묶음이 번갈아 가며 한 번씩 피 터지는 모의고사를 치를 때까지 정확히 $k$ 번 이 짓거리를 돌립니다. 이 눈물겨운 여정이 끝나면, 우리 손에는 $k$ 장의 찢어진 에러 견적 영수증($\text{MSE}_1, \dots, \text{MSE}_k$) 이 쥐어지겠죠. 이 영수증들의 금액을 알뜰하게 평균(averaging) 낸 액수가 바로 위대하신 $K$-폴드 팀의 최종 실전 견적서가 됩니다!

$$
\text{CV}_{(k)} = \frac{1}{k} \sum_{i=1}^k \text{MSE}_i \quad (5.3)
$$

It is not hard to see that LOOCV is a special case of $k$-fold CV in which $k$ is set to equal $n$. In practice, one typically performs $k$-fold CV using $k = 5$ or $k = 10$. What is the advantage of using $k = 5$ or $k = 10$ rather than $k = n$? The most obvious advantage is computational. LOOCV requires fitting the statistical learning method $n$ times. This has the potential to be computationally expensive (except for linear models fit by least squares, in which case formula (5.2) can be used). But cross-validation is a very general approach that can be applied to almost any statistical learning method. Some statistical learning methods have computationally intensive fitting procedures, and so performing LOOCV may pose computational problems, especially if $n$ is extremely large. In contrast, performing 10-fold CV requires fitting the learning procedure only ten times, which may be much more feasible. As we see in Section 5.1.4, there also can be other non-computational advantages to performing 5-fold or 10-fold CV, which involve the bias-variance trade-off. 
자 잠깐! 머리 좀 굴려보면, 방금 전 챕터에서 칭송했던 LOOCV (1명 빼기) 녀석은 사실 이 영리한 $K$-폴드 시스템에 자기 몸집($n$ 명) 전체를 구겨 넣고 $K = n$ 조각으로 미친 듯이 믹서기 썰어버린 변태적 '특수 사례(special case)' 였음을 깨닫기 어렵지 않습니다. 그럼 상식적으로 실전 인더스트리에서는 $K$ 값을 보통 몇으로 돌릴까요? 예외 없이 모두가 짠 듯이 **$K = 5$ 혹은 $K = 10$** 이라는 마법의 파벌 숫자로 플레이합니다. 아니 굳이 $K = n$ 번 풀 파워 전력을 쏟아붓지 않고 고만고만한 $K=5$ 를 고집하는 이기주의(advantage) 가 뭔가요? 가장 소름 돋는 무기는 **연산량 다이어트(computational)** 입니다. LOOCV 는 아까 말했듯 우리를 잔인하게 $n$ 번이나 컴파일하도록 협박하죠(선형 망치 치트키 5.2를 쓸 때만 빼고요). 근데 이 교차 검증은 어떤 최신 딥러닝 텐서 무기든 다 갖다 붙일 수 있는 범용 템플릿입니다. 수십만 개 데이터를 잡아먹고 훈련 1번 도는데 3시간씩 걸리는 GPU 먹는 괴물 무기(computationally intensive) 에다가 $n$ 이 10만 개짜리 LOOCV 를 쑤셔 박는다? 당신 집 전기세는 둘째치고 연산기기가 녹아버리는 치명타(computational problems) 를 맞습니다. 반면! 10-폴드 CV 는 딱 10번만 모델을 굴려주면 깔끔하게 실전 오차를 뽑아내는 미친 가성비 접근(feasible) 을 선사하죠. 게다가 잠시 뒤 5.1.4 스테이지에서 밝혀지겠지만, $K=5$ 나 $10$ 에는 단순 전기세 절감 차원을 넘어 통계학 최고의 피 터지는 난제인 **편향-분산 트레이드오프(bias-variance trade-off)** 를 환상적으로 조준해 치유하는 신비력(비연산적 이점) 이 숨겨져 있답니다.

When we examine real data, we do not know the _true_ test MSE, and so it is difficult to determine the accuracy of the cross-validation estimate. However, if we examine simulated data, then we can compute the true test MSE, and can thereby evaluate the accuracy of our cross-validation results. In Figure 5.6, we plot the cross-validation estimates and true test error rates that result from applying smoothing splines to the simulated data sets illustrated in Figures 2.9–2.11 of Chapter 2. The true test MSE is displayed in blue. The black dashed and orange solid lines respectively show the estimated LOOCV and 10-fold CV estimates. In all three plots, the two cross-validation estimates are very similar. In the right-hand panel of Figure 5.6, the true test MSE and the cross-validation curves are almost identical. In the center panel of Figure 5.6, the two sets of curves are similar at the lower degrees of flexibility, while the CV curves overestimate the test set MSE for higher degrees of flexibility. In the left-hand panel of Figure 5.6, the CV curves have the correct general shape, but they underestimate the true test MSE. 
현생(real data) 에선 전지전능한 신이 아닌 이상 "진짜배기(true)" 테스트 실전 오답 스코어의 전모를 알 길이 없으므로, 우리 CV 교차 검증 녀석이 뱉어낸 견적서가 진짜 예언인지 사기꾼인지 검증할(determine) 방도가 막막합니다. 허나 우리가 신이 되어 직접 데이터 매트릭스를 창조해 낸(simulated data) 가상우주 라면 얘기가 다르죠. 우리는 진짜 테스트 MSE 꿀통을 꿰뚫고, CV 녀석의 시력(정확도 accuracy) 을 심판할 수 있습니다. When we perform cross-validation, our goal might be to determine how well a given statistical learning procedure can be expected to perform on independent data; in this case, the actual estimate of the test MSE is of interest. But at other times we are interested only in the location of the _minimum point in the estimated test MSE curve_. This is because we might be performing cross-validation on a number of statistical learning methods, or on a single method using different levels of flexibility, in order to identify the method that results in the lowest test error. For this purpose, the location of the minimum point in the estimated test MSE curve is important, but the actual value of the estimated test MSE is not. We find in Figure 5.6 that despite the fact that they sometimes underestimate the true test MSE, all of the CV curves come close to identifying the correct level of flexibility—that is, the flexibility level corresponding to the smallest test MSE.
