# 5.1.4 Bias-Variance Trade-Off for k-Fold Cross-Validation
# 5.1.4 왜 하필 K=5, 10일까? 편향-분산 트레이드오프의 비밀!

We mentioned in Section 5.1.3 that $k$-fold CV with $k < n$ has a computational advantage to LOOCV. But putting computational issues aside, a less obvious but potentially more important advantage of $k$-fold CV is that it often gives more accurate estimates of the test error rate than does LOOCV. This has to do with a bias-variance trade-off. 
앞방(Section 5.1.3)에서 우리가 $k$-폴드 CV 녀석을 칭찬했던 가장 큰 이유가 "서버 컴퓨터 불쌍하니까 연산량(computational) 좀 줄이자" 였습니다. 근데 솔직히 말해서, 전기세 아끼는 그런 소박한 이점을 떠나서(putting computational issues aside) 잘 안 알려진 진짜 무시무시한 숨겨진 특권(more important advantage) 이 하나 더 있습니다. 놀랍게도 $k$-폴드 CV 가 그 노무 막노동 끝판왕 LOOCV 보다 오히려 "진짜 실전 에러 점수" 를 훨씬 정교하게(more accurate) 잘 맞춰버린다는 겁니다! 엥? 훈련을 덜 했는데 어떻게 스코어를 더 잘 점치냐고요? 이 기막힌 역전극의 뒤에는 악명 높은 **방패(편향)와 창(분산)의 혈투, 편향-분산 트레이드오프(bias-variance trade-off)** 가 자리 잡고 있습니다.

It was mentioned in Section 5.1.1 that the validation set approach can lead to overestimates of the test error rate, since in this approach the training set used to fit the statistical learning method contains only half the observations of the entire data set. Using this logic, it is not hard to see that LOOCV will give approximately unbiased estimates of the test error, since each training set contains $n - 1$ observations, which is almost as many as the number of observations in the full data set. And performing $k$-fold CV for, say, $k = 5$ or $k = 10$ will lead to an intermediate level of bias, since each training set contains approximately $(k - 1) n/k$ observations— fewer than in the LOOCV approach, but substantially more than in the validation set approach. Therefore, from the perspective of bias reduction, it is clear that LOOCV is to be preferred to $k$-fold CV. 
자, 기억을 더듬어 봅시다. 무식하게 반반으로 잘랐던 초창기 '단순 검증법' 은, 고작 전체 데이터의 50% 반 토막만 훈련소에 입소시켰기 때문에 훈련 부족으로 자신감을 상실시켜 실전 에러율을 심각하게 과대망상 뻥튀기(overestimates) 해버린다고 깠었죠. 이 논리(logic)를 그대로 밀고 가볼까요? 극한의 LOOCV는 단 1명만 빼고 나머지 $n - 1$명(거의 99.9%)을 죄다 훈련시키니까, "진짜 실력(테스트 에러)" 과 비교했을 때 실력이 쫄아들거나 뻥튀기되는 왜곡, 즉 '편향(bias)' 이 제로(unbiased) 에 가깝게 깨끗하게 떨어집니다. 그렇다면 우리가 애용하는 $k = 5$ 나 $10$ 짜리 타협안 $k$-폴드는 어떨까요? 얘는 훈련소에 80~90%의 병력을 밀어 넣으니, 적당히 중간쯤 가는(intermediate) 편향 점수를 얻게 됩니다. (LOOCV보단 훈련량이 딸리지만, 반반 쪼개기보단 압도적으로 많이 공부하니까요). 오! 여기까지 들으면 "편향을 100% 때려잡아서 절대 실력을 왜곡 안 하는 '편향 잡기(bias reduction) 관점' 에선 무조건 LOOCV 가 최강 왕위(preferred) 를 먹어야 하는 거 아님?" 이란 결론이 자연스레 나옵니다.

However, we know that bias is not the only source for concern in an estimating procedure; we must also consider the procedure’s variance. It turns out that LOOCV has higher variance than does $k$-fold CV with $k < n$. Why is this the case? When we perform LOOCV, we are in effect averaging the outputs of $n$ fitted models, each of which is trained on an almost identical set of observations; therefore, these outputs are highly (positively) correlated with each other. In contrast, when we perform $k$-fold CV with $k < n$, we are averaging the outputs of $k$ fitted models that are somewhat less correlated with each other, since the overlap between the training sets in each model is smaller. Since the mean of many highly correlated quantities has higher variance than does the mean of many quantities that are not as highly correlated, the test error estimate resulting from LOOCV tends to have higher variance than does the test error estimate resulting from $k$-fold CV. 
그.러.나! 제발 잊지 마세요. 통계 견적의 세계에서 우리를 괴롭히는 빌런이 '편향(왜곡)' 하나뿐이 아니라는 걸요. 우리는 언제나 주사위가 미친 듯이 널뛰는 **'분산(variance)'** 이란 두 번째 빌런도 때려잡아야 합니다! 그리고 충격적인 반전이 밝혀집니다(turns out). 무적 같던 LOOCV가 사실은 $k$-폴드 녀석보다 이 '분산' 널뛰기 수치가 훨씬 흉악하게 높다는 사실! 아니 왜요(Why is this the case)? 곰곰이 스캔해봅시다. 우리가 극한의 노가다 LOOCV를 돌릴 때, 사실 $n$마리의 복제 모델을 찍어내서 평균을 내는 거잖아요? 근데 이 $n$마리 쌍둥이 모델들이 먹고 자란 훈련소 학생들 명단을 까보면, $n-1$명인데 서로 모델 간에 고작 '1명 차이' 일뿐, 사실상 복붙 수준으로 똑같은(almost identical) 훈련 데이터를 먹고 자란 소름 돋는 클론들입니다. 즉, 이 클론 모델들이 뱉어내는 오답 평가 점수들은 자기들끼리 너무나 징그럽게 끈끈하게 양의 상관관계(highly positively correlated) 로 묶여서, 한 놈이 자빠지면 다 같이 도미노처럼 우당탕 자빠집니다! 반면에 쿨내 나는 $k$-폴드 CV는 어때요? 얘네 모델 $K$ 마리는 서로 간에 훈련소 명단 겹침(overlap) 이 팍팍 줄어들어서, 각자 모델 성향이 꽤나 제각각입니다(less correlated). 자, 통계학 불변의 법칙 하나 쏩니다! "도미노처럼 서로 단합(상관관계)이 쩌는 집단의 평균치 곡선은, 각자 마이웨이 타는 놈들 집단의 평균치보다 지진 난 것처럼 훨씬 심하게 널뛰기(higher variance) 를 한다!" 결국 이 법칙 때문에 LOOCV 의 에러율 견적서는 겉으론 왜곡(편향)이 없어 보여도, 한 번 돌릴 때마다 점수가 우주 끝까지 널뛰는(higher variance) 정신 나간 롤러코스터를 타버립니다!

To summarize, there is a bias-variance trade-off associated with the choice of $k$ in $k$-fold cross-validation. Typically, given these considerations, one performs $k$-fold cross-validation using $k = 5$ or $k = 10$, as these values have been shown empirically to yield test error rate estimates that suffer neither from excessively high bias nor from very high variance. 
자, 피 터지는 대결을 숨 막히게 요약(summarize) 해보죠. 우리가 조각 케이크 $K$ 를 몇 조각으로 썰까 고민하는 건, 사실상 '편향(왜곡 방패)' 과 '분산(널뛰기 창)' 사이의 눈치 게임(트레이드오프 trade-off) 이었습니다. 무수한 고인물 엘리트 해커들이 서버를 태워가며 수십 년간 경험치(empirically) 를 쌓아본 결과, 결국 찾아낸 가장 황금비율의 타협점! 그게 바로 **$K = 5$ 혹은 $K = 10$** 이었습니다. 이 숫자를 박아넣고 돌릴 때, 모델은 심통 부리며 실력을 과소평가(과도한 편향) 하지도 않고, 조울증 걸린 놈처럼 점수가 도톰하게 널뛰지도(극강의 분산) 않는(suffer neither from) 눈물 나게 안정적인 '갓벽한 테스트 예측 타율' 을 선사해 주기 때문입니다!
