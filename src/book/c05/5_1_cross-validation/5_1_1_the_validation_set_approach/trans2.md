# 5.1.1 The Validation Set Approach
# 5.1.1 검증 세트 접근법: 반반 무 많이! 운에 맡기는 모의고사

Suppose that we would like to estimate the test error associated with fitting a particular statistical learning method on a set of observations. The _validation set approach_, displayed in Figure 5.1, is a very simple strategy for this task. It involves randomly dividing the available set of observations into two parts, a _training set_ and a _validation set_ or _hold-out set_. The model is fit on the training set, and the fitted model is used to predict the responses for the observations in the validation set. The resulting validation set error rate—typically assessed using MSE in the case of a quantitative response—provides an estimate of the test error rate. 
자, 우리가 기껏 깎고 다듬은 최신 인공지능 통계 모델이 진짜 실전에서 얼마나 죽을 쑬지(테스트 에러율) 그 견적을 미리 뽑아보고 싶다고 칩시다. Figure 5.1에서 보여주는 _검증 세트 접근법(validation set approach)_ 은 이 골치 아픈 일을 해치우는 방구석 수준의 초단순 작전입니다. 어떻게 하냐고요? 그냥 우리 수중에 있는 데이터 보따리를 눈감고 절반으로 무작위 쪼개기(randomly dividing)를 시전하여 _훈련장(training set)_ 묶음과 _검증 모의고사장(validation set)_ 혹은 짱박아둔 _홀드아웃(hold-out)_ 세트라는 2개의 파벌로 갈라버리면 끝납니다. 모델은 오직 절반짜리 훈련장에서만 피 터지게 싸우며(fit) 근육을 키웁니다. 그리고 훈련이 끝나면, 그 몸짱 모델을 데려와 한 번도 본 적 없는 검증 세트 타겟들에게 첫인상 예측 펀치를 날리게 시킵니다. 그 결과 발생한 오답 헛스윙 점수—연속된 숫자 과녁이라면 보통 오차 제곱합(MSE)으로 매섭게 채점하겠죠—가 바로 우리가 그토록 원하던 "진짜 실전 에러율" 의 대체 견적서가 되어주는 겁니다.

We illustrate the validation set approach on the `Auto` data set. Recall from Chapter 3 that there appears to be a non-linear relationship between `mpg` and `horsepower`, and that a model that predicts `mpg` using `horsepower` and `horsepower`$^2$ gives better results than a model that uses only a linear term. It is natural to wonder whether a cubic or higher-order fit might provide even better results. We answer this question in Chapter 3 by looking at the p-values associated with a cubic term and higher-order polynomial terms in a linear regression. But we could also answer this question using the validation method. We randomly split the 392 observations into two sets, a training set containing 196 of the data points, and a validation set containing the remaining 196 observations. The validation set error rates that result from fitting various regression models on the training sample and evaluating their performance on the validation sample, using MSE as a measure of validation set error, are shown in the left-hand panel of Figure 5.2. The validation set MSE for the quadratic fit is considerably smaller than for the linear fit. However, the validation set MSE for the cubic fit is actually slightly larger than for the quadratic fit. This implies that including a cubic term in the regression does not lead to better prediction than simply using a quadratic term. 
이 무대뽀 반반 쪼개기 작전을 유명한 `Auto` 데이터 주차장에 한 번 써먹어 보겠습니다(illustrate). Chapter 3에서 우리가 `mpg`(연비) 랑 `horsepower`(마력) 사이가 그냥 뻣뻣한 일직선(선형)이 아니라 유연하게 휘어지는 비선형(non-linear) 썸을 탄다는 걸 적발했던 걸 떠올려 보세요. 그래서 1차선 마력 변수에다가 `horsepower`$^2$ 라는 이차선 필살기를 추가한 곡선 모델이 훨씬 찰떡같은 결과를 냈었죠. 사람 욕심이 뻔한지라, "어라? 이차 곡선이 이렇게 좋으면 3차(cubic) 곡선이나 4차, 5차 꼬불꼬불 곡선을 때려 박으면 더 환상적인 예언자가 되지 않을까?" 하고 군침을 흘리는 건 너무나 당연합니다(natural). 우리는 Chapter 3에선 깐깐하게 p-값(p-value) 이란 철퇴 성적표를 들여다보며 এই 과대망상을 짓밟았지만, 이번엔 이 무식하고 직관적인 '검증(validation) 작전' 으로 답을 낼 겁니다. 392개의 자동차 데이터를 절반 뚝 잘라 196대는 훈련 캠프장으로 집어넣고, 남은 196대는 비밀 검증 사격장으로 넘겨버립니다. 훈련 캠프에서 별의별 휘어지는 다항식 회귀 모델들을 달달 볶은 다음, 그 녀석들을 검증 사격장 관측치들에 쏴보면서 MSE 오답 점수를 매긴 성적표가 바로 Figure 5.2의 왼쪽 차트 라인입니다. 확실히 이차(quadratic) U자 곡선 모델의 검증 MSE가 일직선 모델의 오답률을 발라버리면서(considerably smaller) 떡락합니다. 그런데 놀랍게도 3차 꼬불이 곡선의 MSE 성적표는 이차 곡선보다 좋기는커녕 아주 살짝 삑사리가 나며 반등해 버립니다. 이게 뭘 뜻합니까? 모델에 괜히 오버해서 3차선 수학 공식을 집어넣어 봤자 2차선보다 예측 실력이 달리는(does not lead to) 개삽질이란 뼈 때리는 증거입니다.

Recall that in order to create the left-hand panel of Figure 5.2, we randomly divided the data set into two parts, a training set and a validation set. If we repeat the process of randomly splitting the sample set into two parts, we will get a somewhat different estimate for the test MSE. As an illustration, the right-hand panel of Figure 5.2 displays ten different validation set MSE curves from the `Auto` data set, produced using ten different random splits of the observations into training and validation sets. All ten curves indicate that the model with a quadratic term has a dramatically smaller validation set MSE than the model with only a linear term. Furthermore, all ten curves indicate that there is not much benefit in including cubic or higher-order polynomial terms in the model. But it is worth noting that each of the ten curves results in a different test MSE estimate for each of the ten regression models considered. And there is no consensus among the curves as to which model results in the smallest validation set MSE. Based on the variability among these curves, all that we can conclude with any confidence is that the linear fit is not adequate for this data. 
자, Figure 5.2 왼쪽 그림의 선 하나를 긋기 위해 우리가 룰렛을 한 번 돌려 데이터를 무작위 훈련/검증 두 우주로 찢었다는 점을 곱씹어 봅시다. 만약 이 제비뽑기 분할 과정을 제가 심심해서 10번 반복해서 새롭게 반반 무 많이 자르기를 한다면 어떻게 될까요? 뽑기 판이 매번 달라지므로 실전 모델 MSE 견적서의 금액도 매번 고무줄처럼 약간씩 달라질 겁니다. 이걸 적나라하게 까발리는 게(illustration) Figure 5.2의 오른쪽 멀티버스 판입니다. 똑같은 `Auto` 데이터인데 뽑기(random splits) 를 10번 다르게 했더니, MSE 곡선 10가닥이 문어발처럼 제멋대로 춤을 추고 있죠. 물론 이 10마리 문어발들도 공통으로 인정하는 팩트 폭격은 있습니다. 1차 병맛 직선보다는 이차(quadratic) U자 곡선이 극적으로(dramatically) 오답을 줄인다는 것, 그리고 3차 이상의 뇌절 꼬불이 오버 피팅을 해봤자 인생에 하등 쓸모없음(not much benefit)을 10가닥 모두가 시사하죠. 하지만 진짜 개소름 돋는 포인트(worth noting)는, 10번의 제비뽑기 판마다 모델이 뽑아내는 최후의 테스트 에러 견적서(MSE 높낮이 숫자 자체) 가 자기들끼리 심각하게 엇박자를 낸다는 겁니다! 심지어 어느 차수 모델이 왕관(가장 작은 MSE)을 써야 하는지에 대해 이 10가닥 곡선들이 멱살 잡고 싸우며 도통 합의안(consensus) 을 못 내고 있습니다. 이 미친 내적 가변성(variability) 널뛰기 난장판 속에서 우리가 그나마 이마에 힘주고 장담할 수 있는 건 오로지, "아 직선 몽둥이(linear) 로 때리는 건 진짜 최악이다(not adequate)"라는 딱 한 줄의 결론뿐입니다.

The validation set approach is conceptually simple and is easy to implement. But it has two potential drawbacks: 
이 '반반 쪼개기 검증법' 은 뇌를 비우고 직관적으로 이해하기 참 좋고, 코드 한 줄로 썰어버리니 구현(implement)도 혜자스럽죠. 하지만 여기엔 치명적인 잠재적 고질병(drawbacks) 2가지가 잠복해 있습니다:

1. As is shown in the right-hand panel of Figure 5.2, the validation estimate of the test error rate can be highly variable, depending on precisely which observations are included in the training set and which observations are included in the validation set. 
1. Figure 5.2 우측 문어발 환장 파티에서 보셨듯, 이 검증 에러율 견적서는 당신의 '뽑기 운' 에 기하급수적으로 휘둘립니다(highly variable). 어느 금수저 데이터가 훈련소로 납치되고, 어떤 불량 데이터가 검증 사격장으로 유배 갔느냐에 따라 럭비공처럼 튀어 버리는 무시무시한 주사위 던지기 시스템인 거죠.

2. In the validation approach, only a subset of the observations—those that are included in the training set rather than in the validation set—are used to fit the model. Since statistical methods tend to perform worse when trained on fewer observations, this suggests that the validation set error rate may tend to _overestimate_ the test error rate for the model fit on the entire data set. 
2. 게다가 이 무식한 쪼개기 전략은 소중한 관측치 데이터를 반 토막 내어, 오직 절반의 훈련소 그룹만 모델 학원에 입학시켜 공부(fit) 를 시켜버립니다. 불변의 진리! 통계 머신들은 먹여주는 책(데이터) 권수가 적어질수록 머리가 굳어져 시험(성능)을 망치는 고질병이 있습니다. 즉, 절반밖에 공부 못 한 이 억울한 모델이 치른 검증 모의고사 에러율은, 나중에 실전에서 백분율 100% 데이터를 씹어먹고 출전했을 진짜 완전체 모델의 에러율보다 쓸데없이 자신의 약점을 부풀려 _과대평가(overestimate_ 과하게 쫄아붙게 함) 해버릴 지독한 억까 확률이 농후합니다.

In the coming subsections, we will present _cross-validation_, a refinement of the validation set approach that addresses these two issues. 
다음 이어지는 코너에서, 우리는 이 주사위 도박 같은 약점 두 개를 단박에 부숴버리는 반반 검증법의 초진화 모델, 바로 **_교차 검증(cross-validation)_** 의 찬란한 방패술(refinement) 을 꺼내들 것입니다!
